## 0. 生命週期(釋放時序)總覽

```
ComposeSystem (Parse → Module)         建 build-time 託管圖(§B)+ 解析 JSON(_loadedDefs)
   └─ Reset()                          FreeUnmanagedMemory() → 配置全部非託管 hot arrays(§A)
                                        + 從 build 圖攤平出 TransistorList / NodeInfos / NodeTlistGates
                                        + ClassifyPureLogicNodes() (IsPureLogic) + _callbackByNode
LoadSystem()
   └─ ClearPostLoadBuildState()        【hot-path 前清掉 build 圖大物件】Node.Gates/C1c2s、_transistors、
                                        _transistorSet、_forceComputeList、_loadedDefs(JSON,最大宗)
                                        + Gen2 compacting GC。保留 name maps + Node 殼。
[僅 --benchmark 路徑] ReleaseBenchResidualState()
                                        【計時迴圈前最後一掃】清 _nodeByName / _nameByNode / Node 殼 / _nodes
── HOT LOOP(Step / StepCycle)──        只讀 §A 非託管 + §C 常駐託管(_memories / _callbackByNode / palette)
Shutdown() / 下次 Reset()              FreeUnmanagedMemory() → 釋放並 null 掉全部 §A
```

**一句話**:§A(非託管)是 hot data 本身,**不在 hot-path「前」釋放**,只在 `Reset`/`Shutdown` 一次性 free;
§B(build 圖)**在 hot-path 前被乾淨釋放**(`ClearPostLoadBuildState` + bench 再加 `ReleaseBenchResidualState`);
§C(RAM/ROM、callback 表、palette)是 runtime 必要資料,**刻意不釋放**。

## A. 非託管 hot arrays(SoA)— `AllocArray<T>`,64-byte 對齊、配置時清零、登記於 `_allocations`

**它們就是 hot path 的資料,不會「在 hot path 前」被釋放**;hot-path 前不需釋放、也不該釋放。

### `NodeInfo` 單一 item(**16 bytes**,4 個 / 64B cache line;2026-06-02 由 32B 砍半)
**explicit-layout union**:inline payload 與 overflow 的 Tlist 索引**互斥**(所有熱存取都被 `Inline` gate),共用同一塊
12 B(union)。`InlineCap` 7→6 讓 payload = 12 B(= 3 個 Tlist int);`GndCount`/`PwrCount` 壓進一個 byte 的兩個
nibble(各 ≤6),用 property 暴露,所以**熱存取點完全不變**。小扇出(~94%)走 InlinePayload(讀一條 cache line、
不 chase);高扇出/payload≥7(~6%)`Inline==0`、走 Tlist*。bit-exact、interleaved-paired +~0.75%(見 §02 / `02` 報告)。

| offset | 欄位 | 型別 | 說明 |
|---|---|---|---|
| 0 | `Flags` | `byte`(NodeFlags) | 可變:State/PullUp/SetHigh/SetLow/Pwr/Gnd/ForceCompute/HasCallback |
| 1 | `Inline` | `byte` | 1 = 用 InlinePayload;0 = 用 Tlist* |
| 2 | `C1c2Count` | `byte` | inline (gate,other) 對數 |
| 3 | `GndPwr` | `byte` | GndCount = 低 nibble、PwrCount = 高 nibble(各 ≤6;property 存取) |
| 4–15 | **union** | `ushort InlinePayload[6]` **或** `int TlistC1c2s/Gnd/Pwr` | inline 用前者(12 B);overflow 用後三個 int(疊在同一 12 B) |

## D. 重點 / 可再檢視的點

1. **§A 總量 ~1.0 MB(不含 FrameBuffer)**;`NodeInfos`(460 KB)+ `TransistorList`(225 KB)就佔 ~66%。兩者
   都已壓過(NodeInfo inline 省一次 chase;TransistorList 用 ushort 砍半)。L2 = 512 KB/core,所以全表放不進

2. **hot-path 前釋放乾淨?** 是 —— build 圖(含最大宗的 `_loadedDefs` JSON ~20–50 MB)在 `LoadSystem` 即清 +
   Gen2 compacting GC;bench 再清 name maps/Node 殼。計時迴圈只碰 §A + §C。
3. ~~**`_callbackByNode` 是唯一較大的常駐託管陣列**(~115 KB,多為 null)~~ → **已於 2026-06-02 改成稀疏
   `Dictionary`(變更 A)**:省 ~115 KB 常駐 + gen2 GC 不再掃這個參考陣列;bit-exact(full_palette 300k

## E. 實測:把 §C 改非託管值不值得?(2026-06-02)

針對「§C 能否改非託管以加速」實際做了兩個實驗,結論呼應 §D 的分析:**§C 不在決定 hc/s 的熱路徑上**。

- **變更 A —— `_callbackByNode` 稀疏化(`CallbackInfo?[N]` → `Dictionary`)**:✅ **採用**。
  它存的是託管 `Action` 委派,**無法**真正非託管;但改稀疏省了 ~115 KB 常駐 + GC 掃描。bit-exact、非速度改動。
- **變更 B —— RAM/ROM 改非託管存取(pin `byte[]` + 在 hot closure 用 raw `byte*`,隔離 bounds-check 消除)**:
  ❌ **實測後還原**。interleaved-paired(full_palette 300k `--extra-ram`,bit-exact 不變):第一批(未交錯順序)
  +0.54%、第二批(交錯順序、消除 ordering drift)**−0.80%**,B 合計只贏 **8/22** 場 → **落在量測雜訊內,非真實
  增益**。印證 §A/§0 的分析:記憶體存取頻率 ≈ 每 ~14,000 次 recalc 才 1 次,消掉它的 bounds check 對 hc/s 不可見。
  (這也再次示範:sub-2% 的 A/B 一定要 interleaved-paired + 交錯順序,否則 ordering 偏差會給出假的 +0.5%。)

**結論**:§C 的非託管化,只有 A 那種「縮記憶體/GC」的非速度收益值得;為了 hc/s 去動 §C 不值得 —— 瓶頸在 §A 的
BFS,不在這裡。
4. **冷/熱分離已做**:`NodeConnections`(tie-break)、`NodeTlistGates`(寫回)從 `NodeInfos` 拆出,避免污染 BFS
   掃描的 cache line。`RecalcHash`/`_inGroup`/`IsPureLogic` 都是 `byte*`(非 bitset)—— 實測 bitset 的 shift+mask
   反而吃掉 cache 好處。
5. **每次 `Reset` 重新配置 §A + 重建 `_callbackByNode`**;`FreeUnmanagedMemory` 冪等。重跑(`--test-dir` 多 ROM)
   時每顆 ROM 一輪 Reset/Free,無洩漏(`_allocations` 全數 free 後清空)。
6. **§C(RAM/ROM / `_callbackByNode` / palette / 暫存器清單)改非託管,對速度不值得 —— 結論。**
   §C **不在決定 hc/s 的 BFS 熱路徑上**:記憶體存取 ≈ 每 ~14,000 次 recalc 才 1 次、palette 只在渲染時讀、
   `R_Cpu*` 暫存器清單 bench 根本不讀、`_callbackByNode` 只在 rare 的 HasCallback 分支查。實測佐證見 **§E**
   ——把 RAM/ROM 改成 raw `byte*` 存取(消 bounds check)在 interleaved-paired 下是**雜訊(無增益)**,已還原。
   **唯一值得的非託管/瘦身改動是「縮常駐記憶體 + 安靜 GC」型的**(如變更 A 把 `_callbackByNode` 稀疏化),
   **不是為速度**;且 `_callbackByNode` / handler closure 內含託管委派,本就無法真正非託管。**要追 hc/s,動 §A 的
   BFS(NodeInfos / TransistorList 的 cache 行為),不要動 §C。** 這也對應記憶中的 anti-pattern:在非熱路徑 micro-opt。

## F. 實測:縮小 §A 熱陣列(2026-06-02)

兩個「把 §A 縮小」的實驗,interleaved-paired(order-flipped、full_palette 300k `--extra-ram`、bit-exact)。結論:
**縮小有沒有用,看該陣列的 working set 有沒有溢出 cache —— 不是越小越好。**

| 實驗 | 範圍是否夠 | 結果 | 決定 |
|---|---|---|---|
| **`NodeInfo` 32B→16B**(explicit-layout union:payload⊕Tlist 互斥共用 12B;`InlineCap`7→6;Gnd/Pwr 併成 nibble) | node id ushort 夠;Tlist index 需 int(115K)→ 放 union 槽 | NodeInfos 460→**230 KB**;34 輪 **+0.75% TM、22/34 勝** | ✅ **採用**(commit a689a3e) |
| **`RecalcList`/`RecalcListNext`/`NodeConnections` int→ushort** | ✅ 範圍夠(node id ≤14,723;count ≤~53,550;均 <65,535)。**唯 `NodeTlistGates` 不行**(TransistorList index 115K>65,535) | 20 輪 **−0.33% TM、7/20 勝**(微負/雜訊) | ❌ **還原** |

**為何一個賺一個不賺**:`NodeInfos`(460 KB,跨整個 BFS 隨機存取、working set 溢 512 KB L2)砍半 → 更多留 L2 → 省
d-cache miss → +0.75%。而 dirty queue(`RecalcList`)每半週期只用 ~600 筆(~2.4 KB,**本來就在 L1d**)→ 縮成
ushort 沒有 cache 好處,`(ushort)` cast 反而多一點點 → 微負;`NodeConnections` 是冷的(只 floating tie-break)→ 0。

→ **回答「不少 array 可以用 ushort 嗎」:範圍上可以(除了 NodeTlistGates),但只有「大且溢 L2」的陣列縮小才賺;
queue/cold 陣列縮小是中性到微負,不值得。`TransistorList` 早已是 ushort(這個賺,因為它大且熱)。**
