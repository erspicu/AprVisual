# Raspberry Pi 5(ARM / Cortex-A76)測試開發環境 — 工具鏈與登入備忘

2026-06-16 建立。為了補上「第二顆實體微架構」的對照數據:在一台 Raspberry Pi 5 上跑
AprVisual 引擎,取得 ARM(aarch64)的對照數據。本筆記只記**環境 / 工具鏈 / 登入**;實驗數據另記
(見同日「ARM 跨語言性能對照」筆記)。

## 1. 硬體規格(Pi 5)

## 2. 登入設定(SSH 免密碼金鑰)

為了讓 Claude Code 這端能**非互動**(無人值守、不輸密碼)連線,用的是專用 ed25519 金鑰。

| 角色 | 位置 / 內容 |
|---|---|
| 私鑰(Windows 端) | `C:\Users\erspi\.ssh\aprvisual_pi`(**空通關密語**,別外流) |
| 公鑰(Windows 端) | `C:\Users\erspi\.ssh\aprvisual_pi.pub`,comment = `aprvisual-pi-bench` |
| 公鑰(Pi 端已安裝) | `/home/pi/.ssh/authorized_keys` 裡那一行 `...aprvisual-pi-bench` |
| 指紋 | `SHA256:C4UhQD1DhhI8D8MvuKp0f8qfccU0Kqr3tzLhcr5rUeI` |

**連線指令**(從 Windows 的 bash / git-bash):
```sh
ssh -i ~/.ssh/aprvisual_pi -o BatchMode=yes pi@192.168.0.66 '<remote command>'
# 第一次連線曾用 -o StrictHostKeyChecking=accept-new 自動接受 host key;known_hosts 已記住
```
`-o BatchMode=yes` = 金鑰失敗時直接報錯、不退回問密碼(避免非互動卡死)。

**傳檔**:`scp -i ~/.ssh/aprvisual_pi <local> pi@192.168.0.66:<remote>`(Windows OpenSSH 內建 scp)。

**撤銷 / 清乾淨**:刪 Windows 的 `~/.ssh/aprvisual_pi*` 兩個檔,再把 Pi 的
`~/.ssh/authorized_keys` 裡 `aprvisual-pi-bench` 那行刪掉即可。

> 空通關密語的取捨:任何能讀到該私鑰檔的人都能登入這台 Pi。因為是「這台 Windows → 自家區網 Pi」
> 的專用鍵,風險可接受;若要更嚴格就改用有密語的鍵 + ssh-agent(但那樣就無法全自動連線)。

## 4. 部署的目錄結構(Pi 上)

```
~/aprvisual/                          # git clone --depth 1（HEAD c84488d）— 原始碼
  src/AprVisual.S1/                   #   主引擎(NES）
  src/AprVisual.etc/                  #   跨 CPU 工具(--cpu-bench / --naive / --export-*)
    netlists/{6502,6800,z80}/*.js     #   ← scp 補進來的 raw visual6502 網表(gitignore)
  tools/cpp-naive/  naive(+ *.netlist.txt)   # C++ naive；.txt 在 Pi 上由 etc --export-* 重生
  tools/cpp-ours/   ours, ours_full(+ *.engine.txt / *.full.txt)
  tools/visual6502-node/bench.js      #   JS baseline
  ref/visual6502-master/              # ← scp 補進來(整個 ref/ 是 gitignore)
    chipsim.js wires.js macros.js nodenames.js segdefs.js transdefs.js
    chip-6800/  chip-z80/
  run/                                # NES benchmark 用
    data/system-def/...               #   ← scp(網表模組;gitignore)
    full_palette.nes                  #   ← scp(ROM;gitignore)
```

**哪些是 gitignore、必須另外補的**(clone 不會有):
- `src/AprVisual.etc/netlists/<chip>/*.js` ← `scp` 自 Windows 同路徑
- `ref/visual6502-master/`(JS baseline 要的核心 + 各晶片網表)← `scp` 子集
- `run/data/system-def/` + `run/full_palette.nes` ← `scp` 自 `AprVisualBenchMark/`
- C++ 的 `*.netlist.txt / *.engine.txt / *.full.txt` ← **在 Pi 上用 etc `--export-*` 重生**(保證資料一致)

## 5. 建置與執行備忘

### ⚠️ 關鍵雷:csproj 釘死 x64
`AprVisual.S1` / `AprVisual.etc` 的 csproj 寫死 `<Platforms>x64</Platforms>` +
`<PlatformTarget>x64</PlatformTarget>`,在 arm64 上 build 會報 `NETSDK1032`。
**對策:build/publish 一律加 `-p:PlatformTarget=arm64 -p:Platforms=arm64` 命令列覆寫**(不改 csproj,不污染 repo)。

### NES 主引擎(S1)
```sh
# 在 Pi 上 native build(.NET 11 SDK):
cd ~/aprvisual
dotnet build src/AprVisual.S1/AprVisual.S1.csproj -c Release -p:PlatformTarget=arm64 -p:Platforms=arm64
# 跑 benchmark（與桌機 release launcher 同旗標,checksum 可比 golden）:
cd ~/aprvisual/run
~/.dotnet/dotnet ../src/AprVisual.S1/bin/Release/net11.0/AprVisual.S1.dll \
  --benchmark full_palette.nes --bench-hc 400000 --extra-ram \
  --system-def-dir data/system-def --log-dir .
```
> 備用路線:在 Windows 做自包含發佈 `dotnet publish -r linux-arm64 --self-contained`(含 net11 runtime),
> scp 過去直接 `./AprVisual.S1` 跑,完全不依賴 Pi 的 SDK。本機 `temp/pi-arm64/` 留有一份。

### 跨 CPU 對照(6502 / 6800 / z80,NOP-sled)
```sh
ETC=src/AprVisual.etc/bin/Release/net11.0/AprVisual.etc.dll
# 先建 etc(同樣 arm64 覆寫);再產 C++ 匯出:
for c in 6502 6800 z80; do
  dotnet $ETC --export-netlist     tools/cpp-naive/$c.netlist.txt --chip $c
  dotnet $ETC --export-engine      tools/cpp-ours/$c.engine.txt   --chip $c
  dotnet $ETC --export-engine-full tools/cpp-ours/$c.full.txt     --chip $c
done
# 編 C++(g++ 代替 clang):
g++ -O3 -std=c++17 tools/cpp-naive/naive.cpp     -o tools/cpp-naive/naive
g++ -O3 -std=c++17 tools/cpp-ours/ours.cpp       -o tools/cpp-ours/ours
g++ -O3 -std=c++17 tools/cpp-ours/ours_full.cpp  -o tools/cpp-ours/ours_full
# 各引擎跑法（hc 依速度調整,慢的 JS naive 用小 hc;回報的是 hc/s 速率,跨 hc 可比）:
node tools/visual6502-node/bench.js --chip 6502 --hc 6000   --warmup 1000  --rounds 3
dotnet $ETC --cpu-bench src/AprVisual.etc/netlists/6502 --chip 6502 [--naive] \
       --bench-hc 500000 --warmup 50000 --rounds 3 --workload nop
tools/cpp-naive/naive     6502 tools/cpp-naive/6502.netlist.txt 100000 20000 3
tools/cpp-ours/ours       6502 tools/cpp-ours/6502.engine.txt   500000 50000 3
tools/cpp-ours/ours_full  6502 tools/cpp-ours/6502.full.txt     500000 50000 3
```
- C++ / C# 程式 argv:`<prog> <chip> <file> [hc] [warmup] [rounds]`(C++)、
  `--cpu-bench <每晶片netlist目錄> --chip <c> [--naive] --bench-hc/--warmup/--rounds --workload`(C# etc)。
- 量測時間考量:JS naive 只有 ~90–150 hc/s,1M hc 要跑數十分鐘,務必用小 hc(數千)。

## 6. 重點結論(環境層面)

- **位元精確跨 ISA 成立**:S1 NES @400k 在 Pi 上的 checksum `0x9174E19D961CB6E5` 與桌機 golden **一字不差**;
  跨 CPU 三顆的 ours 引擎 checksum 也在 ARM 上自洽(6502 `0x97C10609CF8BF86F` / 6800 `0x713AF51D83FAAC5E`
  / z80 `0x0CE4E8586F898641`,C# ours = C++ ours-core = C++ ours-full)。純整數/位元運算,JIT 在 A76 上產碼一致。
- 工具鏈幾乎全現成(node / g++ 內建,.NET 10 apt 內建),只額外裝了 .NET 11 SDK。
- 唯一固定要記得的雷:**arm64 build 要覆寫 PlatformTarget**;**非互動 ssh 要自己帶 PATH**。
