🔬 方法論與限制

  • 機器:Raspberry Pi 5(Arm Cortex-A76, ARMv8.2, 4c, 主動散熱)。原廠 2.4GHz / 超頻 arm_freq=3000(3.0GHz)兩組都 best-3-of-5、同 400k hc、cpufreq performance governor 鎖定。
  • 超頻設定:/boot/firmware/config.txt 加 arm_freq=3000(原始已備份 config.txt.bak_oc;還原 = 移除該行 + reboot)。2.8GHz→3.0GHz 逐級驗證,每級先確認 bit-exact + 溫度/throttle 才升。
  • 安全:每版測完量溫度,>75°C 即中止(本次最高 37.8°C,未觸發)。零 throttle。
  • 同一 runtime:全部在 .NET 11 上跑(net10.0 版用 DOTNET_ROLL_FORWARD=Major)→ 隔離引擎演進與 runtime 變更。
  • R-1 / S2-A 來自 commit:05.31(R-1)/06.01(S2-A)的 release tag 是 pre-fork WinForms(Linux build 不了),改用引擎 commit a80dab4/9cc0dc7。其餘 16 版由 release tag build arm64。
  • vs x64:見 x64 (Zen2) 報告(末版 ~138.7K)。超頻後 ARM 末版 95,428 ≈ Zen2 的 0.69×。
