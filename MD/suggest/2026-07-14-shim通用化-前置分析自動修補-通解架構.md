# Shim 通用化與前置階段自動修補:通解架構分析

**日期**:2026-07-14
**性質**:架構分析筆記(使用者提問驅動),未排程實作
**兩個問題**:
1. 換別的 NES 遊戲 ROM 有沒有可能踩同樣的雷?現在的 shim 是通用解還是 per-test 解?
2. 能不能用前置階段(載入期)分析抓出語意解析問題的 pattern,自動 emit 修補?

**結論先講**:現有 shim 全是「機制級」而非 per-test,但通用度分三檔,其中「站點限定」
檔換 ROM 確實可能在別的節點踩同類雷;真正的通解不是「一顆 shim 治百病」,而是三層:
**L1 載入期結構偵測 + 自動 emit 修補層(可做,引擎已有一半基建)、L2 引擎語意升級
(強度 LUT / 正準排序,= 已記錄的大爆炸候選)、L3 殘餘校準資料化**。完全零 shim
不可達(A2/A4 的連續時間本質),但「零手寫特例」可達 —— 這就是值得做的貢獻。

---

## 1. 現有 shim 完整盤點(11 顆,依通用度分類)

### (a) 全域物理模型 —— 換任何 ROM 都對,已是通解
| shim | 對應缺陷 | 機制 |
|---|---|---|
| OpenBusShim | B(板卡寄生電容) | open bus = last-transferred-byte 重播,窗 $4020-$7FFF + APU 空間取指 |
| R4015A1Shim | **E(網表資料缺陷)** | 缺失下拉管的 InstClampLow 替身(_ab1→鉗 #10975) |
| PpuWriteDelay | A4(管線延遲) | $2001 寫入延遲 16hc,GLOBAL 窄窗 |

### (b) 機制級、站點限定 —— 物理是通用的,但只鉗了量測過的站點
| shim | 對應缺陷 | 站點 | 同物理的未鉗站點風險 |
|---|---|---|---|
| DlShim | A3(透明閂鎖 capture-once) | 只鉗 $4016/$4017 讀 | **任何** DL 對上任何 turn-on 毛刺的位址;全域版曾因用外部 db 當來源而回歸 $4015 家族 |
| DmcLatchShim | A3+類比賽跑 | pcm_latch 關門沿 | 其他 pass-gate 閂鎖的關門沿 |
| AluLatchShim | A3+類比賽跑 | ALU 輸入閂 | 同上 |
| FrameIrqShim | A2(settle 內暫態) | frame-IRQ flag | 其他 flag ff 吃到暫態 |
| Dbl2007Shim | 類比傳播 | $2007 讀 | 其他 read-buffer 類結構 |
| OamDmaPpuBusShim | 匯流排來源保持 | $4014 DMA 期間 | — |

### (c) 機制級 + 量測校準常數
| shim | 常數 | 來源 |
|---|---|---|
| Dmc4015AbortShim | D=84 ticks、boundary+48t 格點、hold 72t | 對照 AprNes/TriCNES 量測 |
| LxaMagicShim | magic=$FF | 硬體共識值(A1 比例打架的結果) |

**沒有任何一顆認測試名**(per-test 開關已在 IDR 戰役中被明確否決 = 過度擬合)。
但 (b) 檔的「站點限定」是誠實的妥協:同樣的物理在別的節點發生時,現在的 shim
看不見。

## 2. Q1:換遊戲 ROM 的踩雷風險矩陣

先說一個重要事實:**shim 全部是測試模式限定 —— 遊戲/benchmark 路徑現在是「裸」的**,
以下缺陷在遊戲路徑上全部活著:

| 缺陷 | 一般遊戲會不會踩 | 分析 |
|---|---|---|
| E:r4015 缺管 | **會,高頻**| 每個讀 $4017(手把2)的遊戲都在誤清 frame-IRQ 旗標 + 誤觸 status 內部驅動。用 frame IRQ 的遊戲(部分音樂引擎、非 NMI 驅動的老遊戲)會實際壞;多數 NMI 驅動遊戲僥倖無感 |
| B:open bus | **會,中頻** | 手把讀取 `LDA $4016` 的高位就是 open bus($40 來自 $4016 的頁位址殘留);讀未映射區的遊戲(bug 或探測)直接吃錯值 |
| A3:DL capture-once | 會,但站點依 ROM 而定 | 這是 **D 家族樂透**:哪個 DL 對上哪個毛刺由 id-order 決定;新 ROM 的新時序組合可能在未鉗站點觸發(OpenBus err6 的 $4016 好 $4017 壞就是同構站點一好一壞) |
| A2:毛刺時窗 | 精密時序碼才踩 | raster 特效、demo、精確輪詢 $2002 的遊戲風險最高 |
| A4:管線延遲 | 罕見 | 遊戲不需要 cycle-exact 的 $4015 關斷;DMC 重度遊戲(如用 IRQ 計時)有低風險 |
| A1:比例打架 | 罕見 | 用非官方指令 LAE/LXA 的遊戲極少 |

**答案:會踩。**其中 E 和 B 是「每台遊戲都在摸的路徑」;A3 是樂透,換 ROM =
重抽籤。所以「這些修正只影響 AC 測試」的想法不成立 —— 它們是通用正確性問題,
只是 AC 是第一個兇殘到能觀測它們的 ROM。

## 3. Q2:三層通解架構

### L1:載入期結構偵測 + 自動 emit(核心提案,零熱路徑成本)

原則:**語意風險是結構性的,結構可以靜態枚舉。**引擎已有同族基建:fast-path
分類器(3-class、dyn-singleton 偵測)、P-3 load-time prune、tagged gate-list ——
全是「載入期掃結構、跑時零成本」的先例。四個具體自動機制:

**L1-a 透明閂鎖自動偵測 + 透明性重述(最高價值,建議第一步)**
- 偵測 pattern:儲存節點 N,其唯一驅動路徑經過 pass-transistor T,T 的 gate 是
  時鐘相位網(clk1out/clk2out/φ 衍生),否則靠電容保持 —— 純結構條件,一次掃描
  可列出全晶片所有動態透明閂鎖(DL、pcm_latch、ALU 輸入閂……全在此類)。
- 自動 emit:對每對 (閂鎖 N, 相位 P):P 活躍(透明期)時,每半週期 settle 完成後
  檢查 N ≠ settled(輸入側節點) → float-force 回輸入值。
- **這正是 DlShim 的通用化**,且修掉它全域版失敗的根因:當年全域版錯在把「外部
  db」當重述來源($4015 讀是內部的,外部是垃圾);自動衍生版天然使用**該閂鎖
  pass-gate 的實際輸入節點**,語意永遠對。
- 預期效果:DL/DMC-latch/ALU-latch 三顆手寫 shim 收斂成一個自動機制,並自動覆蓋
  所有「還沒被測試打到」的同構站點 → 直接回答 Q1 的樂透風險。
- 成本:載入期一次掃描;跑時每半週期 K 次 byte 比較(K=偵測到的閂鎖數,估計
  數百~數千)→ 先掛測試/精確模式量測,無回歸再談常開。
- 風險:誤偵測(把非閂鎖當閂鎖強制)→ 需要白名單/審計輸出;與 under-settle
  教訓無衝突(這是 post-settle 覆蓋,不是截斷 —— DlShim 註解實測「正確週期上
  是 no-op」)。

**L1-b 動態儲存節點的 settle 暫態硬化**
- 偵測:同 L1-a 的儲存節點集合,加上 flag ff(交叉耦合對)。
- 規則:settle 過程的中間值不得成為儲存節點的最終捕捉值 —— 捕捉點只認 settle
  完成後的穩態(FrameIrqShim 的通用化)。
- 難點:「捕捉發生在 settle 內」是事件不是狀態,通用攔截需要在 SetNodeState 層
  掛鉤 → 有熱路徑成本疑慮,價值/成本比低於 L1-a,列第二梯隊。

**L1-c 多驅動衝突站點普查(bus-fight census)**
- 靜態列出所有「兩個以上不同控制錐的驅動源可同時導通到同一群組」的站點
  (A1 家族的完整地圖)。不自動修 —— 產出審計報告 + 站點清單,供 L2 強度
  解析或人工建模。純離線工具,零風險。

**L1-d E 類獵人:網表 vs 獨立矽參考的自動審計**
- 本次 29 線 PLA 審計的工具化:transdefs 乘積項解剖 → 對照 BreakNES 模型
  (regs.cpp 的 PLA 表是機器可讀的)→ 差異報告。
- 修補格式:**載入期修正層資料檔**(netlist-patch.json:「加一顆管 gate=X c1=Y
  c2=vss,理由=BreakNES pla[4]」)—— 資料與程式分離,与類比 Sidecar 藍圖同構。
  這不是特例解:格式通用、內容資料化、每筆帶獨立來源佐證。

**L1-d′ 幾何完備性審計(2026-07-14 追加;自洽檢查,不需外部參考)**
- 依據:兩個已知洞(MetalNES 補的 t14634b、我們的 r4015-a1)經頂點級驗證
  **幾何皆完整存在於 segdefs**(poly 條帶 + 兩側異節點擴散 = 必然是電晶體),
  只是 transdefs 萃取漏發 —— 同一萃取腳本的系統性漏判,沒理由只漏兩次。
- 演算法:枚舉 segdefs 所有「poly×雙側擴散」交叉 → 對照 transdefs 器件表 →
  無對應列者 = 候選洞 → BreakNES / 硬體行為複核(產報告,不自動修)。
- 範圍:2A03 + **2C02(從未被審過)** —— 剩餘未修的 PPU 側三顆
  (BGSerialIn / StaleSpriteShiftRegs / Address2004)有機會其實是 E 類。
- 界線:只能抓「萃取漏」;描圖級漏洞(照片→多邊形就缺)不可見,仍需外部參考。
- **2026-07-14 已執行(tools/netlist/geom_audit.py)**:校準 = 在原始 2A03 上以
  精確 die 座標重抓兩個已知洞 ✓;判決 = 2A03 功能有效萃取漏**恰好 2**(皆已補)、
  **2C02 = 0**。此缺陷類別在兩顆晶片上就此封閉;剩餘五顆未修測試的 E 類嫌疑
  大幅降低,回歸 A/B/D 預期。

### L2:引擎語意升級(opt-in 精確模式;= 已記錄的大爆炸候選)

| 升級 | 治的缺陷 | 狀態 |
|---|---|---|
| 強度類別 LUT(MOSSIM II 式,面積電容代理) | A1 全家(LAE/LXA 免 shim 自然解出) | 大爆炸候選(checksum 必變),已記錄 |
| 正準排序(群組解析與 id-order 解耦) | D 全家(樂透從根消滅) | 大爆炸候選,已記錄 |
| 時戳類比層(Analog Sidecar) | B 全家(open bus 衰減、OAM decay、RC) | 藍圖已立(2026-07-14 混合訊號筆記) |

這三件事共同構成「Accuracy Epoch」:一次打包、金 checksum 重定基準、全量重驗。
單獨做任何一件都要重擲樂透,所以打包比逐件划算。

### L3:無法靜態化的殘餘 —— 校準資料化

A4(管線延遲)的本質是連續時間,靜態分析給不出 D=84 這種數字(需要 RC 延遲
模型才能推導,那是 SPICE 級)。誠實方案:保留機制級校準 shim,但把校準常數
從代碼移進 sidecar 資料檔(`timing_calibrations.json`:機制、節點、常數、量測
來源、複驗方法)。手寫的是「機制框架」,長的是「資料」。

## 4. 誠實的極限

- **A2/A4 無完全通解**:離散 settle 模型沒有奈秒;完全消滅需要時序標註模擬
  (每閘延遲)或 SPICE —— 成本數量級不可接受(見 MOSSIM II 對照筆記 §7)。
- **L1-a 覆蓋的是「閂鎖側」**:毛刺產生端(A2)不動,只把「捕捉端」硬化 ——
  毛刺若被非閂鎖結構消費(組合邏輯瞬間分支)仍可能漏。
- **零 shim 不可達;零手寫特例可達。**目標函數:手寫 per-site 代碼行數 → 0,
  換成「自動衍生機制 + 資料驅動修正/校準檔」。

## 5. 建議路線圖(全部排在掛牌 bank 之後)

1. **P0(離線,零風險,隨時可做)**:L1-c bus-fight census + L1-d E 類獵人
   工具化 —— 純分析腳本,產報告不動引擎;E 類獵人順便把 2C02 的解碼也掃一遍
   (2A03 都漏一顆管了,PPU 沒理由更乾淨)。
2. **P1(測試模式原型)**:L1-a 透明閂鎖自動偵測 + 重述,先跑 AC 全套 +
   哨兵對照:預期 DL/DMC-latch/ALU-latch 三顆 shim 可退役、若干未破測試
   (Address2004、StaleSpriteShiftRegs?)可能免費翻綠 —— 這是最快能驗證
   「自動修補」概念的實驗。
3. **P2(Accuracy Epoch 大爆炸)**:L2 三件套 + E 類 netlist-patch 常開 +
   L1-a 常開,金 checksum 重定基準,全量重驗(AC 141 + testrom 全家族 +
   遊戲畫面 CRC 語料庫),需使用者拍板。
4. **驗證閘不變**:AprNes 差分神諭、isolated ROM 對表、interleaved-paired
   效能(常開機制必須證明 ≤2% 成本)。

## 6. 一句話總結

現在的 shim 是「機制級、站點限定、測試模式」的過渡態;通解 = 把「站點限定」
升級成「結構自動枚舉」(L1),把「機制手寫」升級成「引擎原生語意」(L2),
把「常數埋碼」升級成「校準資料檔」(L3)。第一塊試金石是透明閂鎖自動重述 ——
它同時是三顆現有 shim 的收斂點、D 家族樂透的部分解、和「前置分析自動修補」
概念的最小可驗證原型。
