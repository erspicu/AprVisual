RESPONSE
========================================================================
To address the extreme sensitivity of the C#/Rust JIT/AOT compilers to branch-prediction and instruction-cache density, we must change how we think about adding fast-paths. 

Constraint #1 (branch-cost asymmetry) is your biggest enemy. If you add a new classification by adding an `else if (node.IsClassX)` to `recalcNode`, you penalize the 26.7% fast-path and the remaining fallback path. 

**The Enabler: Multi-Queue Static Dispatch**
Before introducing new static classes, you must decouple their execution paths. 
Allocate separate dirty-node FIFOs (or a single array of separate queues) for each static class: `queue_fast`, `queue_isolated_pair`, `queue_full`. 
During `setNodeState`’s fanout walk, you enqueue by static type: `queues[target_node.type].push(target_node)`. 
Your settle loop becomes:
```csharp
while (queues_have_work) {
    while (queue_fast.TryDequeue(out nn)) recalc_fast(nn);
    while (queue_pair.TryDequeue(out nn)) recalc_pair(nn);
    while (queue_full.TryDequeue(out nn)) recalc_full(nn);
}
```
**Why this works:** The enqueue step pays a negligible data-dependency cost (reading the static type). The dequeue/recalc step is now **branch-free regarding node classification**. LLVM and RyuJIT will heavily inline and unroll these homogenous loops. This entirely defeats Constraint #1, allowing you to add specialized paths without bloating the existing fast/full paths.

Here are the new static classifications and lowering strategies, ranked by expected ROI against your memory-latency and branch-prediction bottlenecks.

### PART 1: STATIC NODE-CLASSIFICATION STRATEGIES

Given the average walk is ~1.4 nodes, most of your BFS time is spent setting up and tearing down the `_inGroup` array and queue for trivially small topologies.

#### 1. Isolated Bounded Components (The "Size-2/3 Island")
Many pass-transistors in NMOS/CMOS exist in isolation (e.g., a transmission gate connecting a driver to a latch, or an isolated half-latch). 
*   **Static Property:** At reset, compute the connected components of the graph using *only* `c1c2` edges. Identify components of exactly size 2 (Nodes A and B) or size 3. These nodes have `TlistC1c2s > 0`, but their pass-transistor network is strictly closed.
*   **O(1) Resolution (for Size 2):** 
    ```csharp
    // Inside the queue_pair drain loop. nn is known to be in a closed {A,B} pair.
    Node other = nn.StaticPairOther;
    bool gateOn = NodeStates[nn.StaticPairGate] != 0;
    
    byte flags = nn.Flags;
    flags |= (gateOn ? other.Flags : (byte)0); 
    // evaluate pwr/gnd scans for nn (and other, if gateOn) via loop-unrolled checks
    // ...
    // Floating tie-break: if (flags == 0), value = (nn.Cap > other.Cap) ? nn.prev : other.prev; 
    ```
*   **Bit-Exact Proof:** Because neither A nor B has `c1c2` edges to any node outside {A,B}, the theoretical BFS group starting from either node is mathematically bounded to exactly {A}, {B}, or {A,B}. The unrolled logic evaluates this exact set without queue allocations or `_inGroup` writes. The capacitance tie-break strictly matches the BFS behavior.
*   **Coverage & Savings:** Expected to cover ~10-15% of nodes. Saves the `_inGroup` clear/check, queue bounds checking, and array indirections. Drops D-cache hits on the BFS queue/dedup arrays to zero for these nodes.

#### 2. The "Driven-Stub" Leaf Node
*   **Static Property:** A node `L` (Leaf) where `TlistC1c2s` has exactly 1 entry pointing to `P` (Parent). `L` has NO `TlistC1gnd`, NO `TlistC1pwr`, NO PullUp, and NO `ForceCompute/HasCallback`. 
*   **O(1) Resolution:** When `L` is dirtied, do NOT compute it. Just clear its dirty flag. Enqueue `P` instead. 
*   **Bit-Exact Proof:** Since `L` has no independent ability to drive its state (no gnd/pwr/pullup), its state is 100% dictated by `P` whenever the pass-transistor is ON. If the transistor is OFF, `L` is purely floating and holds its previous state. By deferring to `P`'s recalculation, `P`'s normal BFS will absorb `L` (if ON) and update L's state. If `P` evaluates and the transistor is OFF, `L` is correctly left floating. *Note: Fanout from `L` is perfectly preserved because when `P` updates `L`, `setNodeState(L)` still fires L's fanout.*
*   **Coverage & Savings:** Expected ~5%. This completely eliminates the BFS origin from useless stub nodes, folding the work into the parent's existing recalculation.

### PART 2: LOWERING STRATEGIES (Pre-computation / Canonicalization)

Given your PMC data showing D-cache misses at 5.09% per cycle are the dominant stall, your lowering pass MUST prioritize memory layout and graph compaction over instruction count.

#### 1. Hot/Cold Struct Splitting (AoS to SoA for L1d Budget)
Your L1d budget is ~32KB. `NodeStates` (14KB) + `_inGroup` (14KB) = 28KB. Your 16-byte `NodeInfo` array is ~240KB. Every time the BFS touches `NodeInfo`, it evicts critical state lines.
*   **Transformation:** Split `NodeInfo` into two separate arrays. 
    *   `HotNodeInfo` (BFS data): `Flags`, `TlistC1c2s`, `TlistC1pwr`, `TlistC1gnd`. (Pack this into 8 bytes).
    *   `ColdNodeInfo` (Fanout data): `tlist_gates`.
*   **Why it's bit-exact:** Semantics are untouched, just memory layout changes.
*   **Locality Gain:** The BFS loop *never* reads fanout. Fanout is only read in `setNodeState` *if the node state actually toggles*, which is a fraction of the time. Halving the hot struct size from 16 to 8 bytes doubles your L1d cache density during the memory-bound BFS, significantly dropping the 5.09% miss rate.

#### 2. Supply-Anchored Merge (Answering your specific question)
*   **Static Rule:** If a normal node `N` is connected to VCC or GND via an unconditionally ON (gate == VCC), NON-weak transistor.
*   **Is it bit-exact?:** YES. 
*   **What about its capacitance tie-break?:** It does not matter. Because `N` is hard-tied to a supply via a non-weak transistor, `N` can *never* enter a floating state. Any conducting group containing `N` will inherit `N`'s supply connection. The `getNodeValue()` LUT prioritizes Gnd/Pwr over floating. Therefore, `N`'s capacitance is mathematically unreachable logic.
*   **Transformation:** 
    1. Delete `N`.
    2. For every `c1c2` transistor connecting `N` to `other` (gated by `G`), move it to `other`'s `TlistC1pwr` (if N was VCC tied) or `TlistC1gnd` (if N was GND tied), gated by `G`.
    3. For every fanout gate `N` drove, statically replace its gate state with 1 (VCC) or 0 (GND). This will likely identify dead transistors (if GND) or newly always-on transistors (if VCC), allowing cascading lowering passes.
*   **Coverage & Gain:** Moderate coverage, but massive impact per node. It converts O(n) BFS step traversals into O(1) early-break `pwr/gnd` scans for all of `N`'s neighbors.

#### 3. Graph-Locality Renumbering (Targeting D-Cache Stalls)
*   **Static Rule:** Renumber all Node IDs (0...14,729) so that nodes frequently co-visited in the same BFS land on the same 64-byte cache line. 
*   **Algorithm:** Use Reverse Cuthill-McKee (RCM) or a greedy BFS layout. 
    *   Edge Weight 10: `c1c2` pass-transistors (traversed on every BFS).
    *   Edge Weight 2: `fanout` gates (traversed on toggle to enqueue).
    *   Start the layout BFS from the most heavily connected nodes (e.g., clocks, prominent bus lines).
*   **Why it works:** `NodeStates` is densely packed (1 byte = 1 node). 64 nodes fit in a single D-cache line. If a pass-transistor connects Node 5 to Node 8000, traversing it costs 2 cache misses. If they are renumbered to Node 5 and Node 6, they share a cache line. Your 1.4 node average walk will almost always hit L1d if neighbors are contiguous in ID space.
*   **Risk:** Zero risk to correctness. Purely an index mapping applied at parse-time.

#### 4. Parallel Pass-Transistor Compaction
*   **Static Rule:** If two normal nodes A and B are connected by *multiple* `c1c2` pass transistors (e.g., `T1(gate=G1)` and `T2(gate=G2)`).
*   **Transformation:** You cannot easily OR gates dynamically without adding a node. However, if `G1 == G2` (redundant parallel transistors), drop one. If `G1` is the inverse of `G2` (e.g., a CMOS transmission gate but one side is statically always inverse of the other), keep them but flag the pair for the "Size-2 Island" (Part 1, Idea 1) since you only need to evaluate if *either* is ON.
*   **Gain:** Minor graph shrinkage, but cleans up artifacts from raw die extraction.
