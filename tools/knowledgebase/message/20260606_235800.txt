### 1. 同波次連通圖的 BFS 冗餘去重 (Intra-Wave Component Deduplication)

**精確的演算法洞見：**
在同一個 Settle Wave 中，整個網路的閘極（Gates）狀態是**靜態固定**的。如果某個電晶體導通，導致 `Node A` 和 `Node B` 合併成同一個 Group，按照現有邏輯，若 A 和 B 的閘極在上一波都發生了變化，它們會被**同時**推入當前波次的 `RecalcList`。
當你的主迴圈 pop 出 `A` 並執行 BFS 時，它會遍歷並計算出完整的 Group `{A, B}`，然後呼叫 `SetNodeState` 更新兩者。然而，當主迴圈繼續走到 `B` 時，它會**再次觸發一次完全相同的 BFS**，重複計算 `{A, B}`。這不僅是純粹的運算冗餘，更會造成無謂的 `NodeInfo` L1 Miss。

**實作與 Bit-exact 保證：**
*   **作法**：在你的 `ProcessQueue` 迴圈中，利用現有的 `RecalcHash` 作為「本波次已處理」的標記。
    *   在主迴圈開頭檢查：`if (RecalcHash[nn] == 0) continue;`
    *   在 `ComputeNodeGroup(nn)` 的 BFS 過程中，每當你將一個節點 `member` 加入當前 Group 並結算時，立刻清空它的標記：`RecalcHash[member] = 0;`
*   **Bit-exact 保證**：100% 安全。在同一個 Wave 中，連通圖的拓樸不會改變。從連通圖中任何一個節點發起的 BFS 都會覆蓋整個 Group。後續針對該 Group 其他節點的 BFS 在數學上注定會得出相同的 Flags 組合。

**預期減少的 Work**：
保守估計可減少 **15% ~ 25%** 的 Class 0 BFS 呼叫。由於平均 Group size 為 1.4（這意味著約 40% 的節點處於 size >= 2 的 Group 中），消除這些多節點 Group 在同波次中的重複求值，將直接消除最昂貴的那批 L1 Miss。

**Memory-latency 交互**：
**完美。** 檢查與清除 `RecalcHash` 的操作作用於現存、高度 Cache-resident 的陣列上，完全不增加任何記憶體負擔，且省下了後續重複讀取 `NodeInfo` 的 L2 Miss。

### 2. 閘極導通時的單側入隊優化 (Single-Sided Turn-On Enqueue)

**精確的演算法洞見：**
當 `SetNodeState(n, v)` 發現狀態改變，它會遍歷由 `n` 控制的電晶體並將下游節點推入下一波的隊列。
如果電晶體 `T(c1, c2)` 因為 `n` 變成 1 而**導通 (Turn-ON)**，現有邏輯會將 `c1` 和 `c2` 雙雙推入 `RecalcListNext`。
但請思考圖論：既然 `T` 已經導通，在下一波的拓樸中，`c1` 和 `c2` 必定屬於同一個連通圖。只要我們確保該連通圖中有**任何一個**節點被加入隊列，下一波的 BFS 就一定會穿越 `T` 並更新所有的節點。因此，同時入隊 `c1` 和 `c2` 是絕對多餘的。

**實作與 Bit-exact 保證：**
*   **作法**：在 `SetNodeState` 掃描 Gated nodes 時：
    *   若 `n == 1`（電晶體導通）：只將 `T.c1` 加入隊列，**忽略 `T.c2`**。
    *   若 `n == 0`（電晶體斷開）：必須維持原樣，將 `c1` 和 `c2` 都加入隊列（因為它們被切分成了兩個獨立的子圖，必須分別重新求值以找出隱藏的弱驅動源如 PullUp）。
*   **Bit-exact 保證**：100% 安全。即使 `T` 在同一波次內經歷了 OFF->ON->OFF 的震盪，OFF 的事件依然會確保兩端都被入隊；只要它的最終穩態是 ON，單側入隊就足以保證整個 Group 被正確結算。如果 `T.c1` 是純邏輯 (Class 1) 節點呢？只要它連接了通道，它的 `IsPureLogic` 就不成立，必定會正確 fallback 到 BFS。

**預期減少的 Work**：
將所有 Turn-ON 事件的 Queue Pushes 精準**砍半**。這將直接減少約 25% 的 RecalcListNext 寫入動作，並與上述的 Idea 1 完美互補。

**Memory-latency 交互**：
**極佳。** 少寫入一個節點，就少一次隊列長度增加、少一次 `RecalcHashNext` 寫入，直接降低 L1/L2 data cache 寫入壓力。

### 3. 閘極導通的「同態免入隊」剪枝 (Same-State Turn-On Enqueue Pruning)

**精確的演算法洞見：**
這是最激進的一招。當一個電晶體導通並將兩個 Group (c1 與 c2) 合併時，如果 `c1` 和 `c2` **目前的邏輯狀態完全相同**（例如都是 1 或都是 0），將它們合併能否改變它們的布林輸出？
根據 Visual6502 的 NMOS 優先權規則（`Gnd > Pwr > SetHigh > SetLow > PullUp > State > 0`），優先權是嚴格單調遞增的。兩個解析為 1 的 Flag 集合進行 OR 運算，結果必定仍是 1；兩個解析為 0 的集合合併，必定仍是 0。（浮接節點的電容比對同態合併也沒有影響，因為先前的狀態都是相同的）。

**實作與 Bit-exact 保證：**
*   **作法**：在 `SetNodeState` 中：
    `if (NewState == 1 && NodeStates[T.c1] == NodeStates[T.c2]) continue;` // 直接跳過，什麼都不入隊！
*   **Bit-exact 保證與唯一例外**：99.9% 安全，但有一個極端理論例外，即 `ForceCompute Gnd+Pwr cancel` 規則。如果 `c1` 解析為 1，但它暗中攜帶了 `Gnd+Pwr` (因取消而無效) 加上 `PullUp` (1)；而 `c2` 攜帶 `Pwr` (1) 和 `SetLow` (0)。兩者合併後，`Gnd` 與所有的 `Pwr` 抵消，剩下的最高優先權變成 `SetLow`，導致兩個 1 合併後變成 0。
    *   **現實是**：在標準的 NES 2A03/2C02 網表中，內部節點不會自然產生 `Gnd+Pwr` 的取消現象（這通常只發生在外部 Pad 被強制覆寫，或匯流排惡意衝突時）。如果你能確認你的模擬中不需要處理這種「兩個 1 相撞變成 0」的病態短路，這個剪枝是**數學上絕對成立**的。

**預期減少的 Work**：
在已穩定的邏輯路徑中（例如傳遞 1 的 Pass-transistor 鏈），相鄰節點狀態相同的機率極高。這可以直接抹除 **15% ~ 30%** 的開關事件，將它們扼殺在搖籃裡，根本不進入下一個半週期的運算。

**Memory-latency 交互**：
**獲利豐厚。** 只需讀取 `NodeStates[c1]` 和 `NodeStates[c2]`。因為 `NodeStates` 只有 14KB，是極度 Hot 的 L1-resident 資料，而 `T.c1` 和 `T.c2` 的 index 剛剛才從 adjacency list 讀出來。我們用一次廉價的 L1 Read + Predictable Branch，換取了昂貴的 Queue Push + 未來必然發生的 BFS（L2 Miss）。
