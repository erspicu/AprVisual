## 1. 願景:為什麼要做混合訊號(Mixed-Signal)模擬

整台 NES 除了 2A03/2C02 兩顆數位晶片外,主機板是一張充滿電阻、電容、電晶體與
音效濾波迴路的**類比電路板**。目前引擎撞上的 OAM 衰減、Open Bus 電容殘影等現象,
本質都是「數位邏輯碰上類比物理世界」的邊界效應。

目標型態:**數位側維持現有 switch-level 黃金引擎(單執行緒、settle-to-quiescence),
類比側把少數「弱節點」與板載類比零件抽成極輕量的時域模型**,兩側以事件驅動的
動態邊界銜接(Mixed-Signal Co-Simulation)。做成之後,專案從「遊戲模擬器」升格為
「兼具數位邏輯與類比物理特性的 NES 矽晶片虛擬博物館」。

**絕對禁止的路線**:把整機丟進 SPICE 解全域微分方程矩陣 —— 速度會掉到幾 Hz 等級,
連一幀都跑不完。

## 2. 核心機制:時戳惰性評估(Timestamp-based Behavioral Analog)

處理電容衰退(如 OAM DRAM 漏電)不需要在熱迴圈算微積分,而是:

1. **載入期預計算** —— 對已知的動態閂鎖/電容節點註冊放電特徵常數(τ=RC)。
   目標只是要知道「何時跌破邏輯閾值 V_th」,把指數公式反解成固定整數週期數:

   `Δt_drop = −RC · ln(V_th / V_0)` → 載入時算一次,存成「斷開後 N 個 cycle 翻 0」。

2. **斷開時戳** —— 數位側走訪發現節點驅動源斷開(進入 Floating)時,
   只做一次整數賦值:`node.DisconnectTimestamp = CurrentCycle`。

3. **結算點查帳** —— 節點下次被讀取或參與 charge-sharing 時:
   `if ((CurrentCycle - ts) > DecayThresholdCycles) state = 0;` —— 一次整數減法 + 比較。

### 效能帳

| 做法 | 成本 |
|---|---|
| 每半週期算 `V(t)=V0·e^(−t/RC)`(SPICE 模式) | **毀滅性** —— 浮點 + Math.Exp 進熱迴圈,速度跌破 1 kHc/s,絕對禁止 |
| 時戳惰性評估 | 預期 **0% ~ −2%**;純整數、無遮罩、不破壞 cache locality |

不重演 P-5「維護成本倒賠」悲劇的三個理由:
- 影響範圍極小:1.47 萬節點中需監控的邊界節點(OAM 記憶單元、Data Bus 等)佔比極小;
- 是**替換**既有行為層 if-else/計數器,不是疊加新成本,一來一往可能更快;
- 只有整數加減比較,無浮點無位元遮罩稅。

> 專案既有原則呼應:「加成本必輸,唯移除成本才贏」(hotpath 天花板總綱)。
> 此機制屬於「載入期靜態事實 + 結算點惰性查帳」,與 P-3 load-time prune 同族。

## 3. 行為層 Hack ↔ 類比物理真身對照表

現有(或典型模擬器的)行為層代碼,未來要被類比 Sidecar 替換的佔位符:

| 行為層模擬(Hack) | 物理真身 |
|---|---|
| `last_data_bus_value` + 生命週期後清空(Open Bus 殘值重播) | PCB 銅箔走線寄生電容:Hi-Z 浮接時殘留電荷慢慢放電 |
| 開機/Reset 後 CPU 延遲 N 週期才跑 | RESET 腳位的 RC 延遲電路:電解電容充電數十毫秒爬過 V_th |
| 渲染關閉時 OAM 資料按計數器爛掉 | 2C02 OAM 是 DRAM = 幾千顆微小電容,無 refresh 就漏電 |
| NMI/IRQ 延遲半個~一個 cycle 才被偵測 | 腳位上拉電阻 + 線路電容:0V→5V 充電爬坡時間差 |

演進路徑:現階段留著行為層(標 `// TODO: Replace with RC decay physics`),
未來收斂成統一物理法則 —— 不再寫 `if (OpenBus)`,而是給資料線節點定義 0.1pF 電容;
不再寫 Reset Delay,而是給 RESET 節點定義 RC=30ms 充電方程。

## 4. 參數與資料來源

### 4.1 板載標準零件(BOM)

以 NTSC 北美版 **NES-CPU-04** 主機板為例(型號皆**待查證**):

- CPU/APU:理光 RP2A03;PPU:理光 RP2C02
- CPU RAM / PPU VRAM:2KB SRAM(Sanyo DM2016 或 Toshiba TC5516AF)
- 主時脈石英:21.47727 MHz(CPU ÷12、PPU ÷4)
- 視訊放大電晶體:2SC2021 或 2SC1740(NPN 小訊號)
- 電源:LM7805
- **音效混音高通濾波電容:C21(1µF 50V)、C22(10µF 16V)** —— 決定音效衰減與低音截止頻率
- 視訊去耦:C23(220µF 10V)

最佳測繪來源(**待查證**):Leon Kiriliuk 的 **NES-CPU-04 互動式電路圖專案**
(全板類比走線/電阻/電容/電晶體以現代 EDA 重繪,含 CIC 晶片)—— 直接讀 BOM 數值。

主動元件的 SPICE Model(.lib/.mod):製造商官網或開源 LTspice/Ngspice 零件庫
(GitHub 搜 `LTspice standard.bjt`),內含 Cbe、Bf 等物理特性,純文字可直接 parse。

### 4.2 晶片內部「隱藏類比零件」

- **節點電容值**:Visual6502 逆向時掃描實體光罩多邊形;開源庫中有衍生資料
  「基於多邊形面積計算的各節點預估電容值」(生成網表的 Python 腳本內,segdefs/node
  擷取相關目錄;面積越大寄生電容越大)。(**待查證** —— 注:另一份諮詢已把
  「多邊形面積電容代理」列為大爆炸候選:checksum 必變,採用前須離線工具先驗。)
- **漏電流**:無官方數據(取決於 1980s NMOS 製程與溫度),實務上由測試 ROM 實測
  **反推**:如 `oam_decay.nes` 測得約 30ms 損壞 → 反解 Decay Threshold 該設幾個 CPU cycle。

## 5. 描述格式的選擇

| 格式 | 定位 | 適用性 |
|---|---|---|
| SPICE Netlist(.cir/.sp) | 類比電路的全球公認格式,純文字極簡(`R21 節點A 節點B 2.2k`) | 外圍 RC 拓樸可直接 Regex parse 載入 |
| Visual6502 JSON(segdefs/transdefs) | 逆向工程界的平鋪網表事實標準 | 本專案底層現行格式 |
| Verilog-AMS / VHDL-AMS | EDA 工業界數位+類比混合標準,可同檔寫邏輯與微分方程 | 太重工業,需百萬級 EDA 軟體,不可行 |
| **自訂 Sidecar JSON(建議)** | 輕量副網表標註檔,擴充現有電晶體清單 | **最佳解**:資料與程式分離、不動熱迴圈 |

Sidecar 範例(`analog_properties.json`):

```json
{
  "dynamic_latches": [
    { "node_id": 1042, "type": "OAM_DRAM_CELL", "capacitance_pf": 0.05, "decay_cycles": 166000 },
    { "node_id": 2155, "type": "BUS_DECAY",     "capacitance_pf": 0.12, "decay_cycles": 8000 }
  ],
  "pcb_filters": [
    { "name": "C21", "type": "CAPACITOR", "value_uf": 1.0, "connect_from": "apu_mix", "connect_to": "audio_out" }
  ]
}
```

載入時把有類比屬性的 node_id 打上 `IsAnalogDependent` 靜態旗標 +
輔助陣列存預算好的 DelayCycles;熱迴圈只剩整數時戳比對。

### APU 音效與視訊濾波

2A03 音頻輸出是電流,板上以電容 + 運放做高通/低通濾波。實作:在引擎外圍掛
輕量 IIR/FIR 數位濾波器網路,把 switch-level 算出的 DAC 輸出位元丟進去 ——
可還原類比零件老化與不同板版(前期機 vs 後期機)的音色差異。

## 6. 最難的骨頭:反向因果(Analog→Digital Feedback)

類比電壓衰減到一半時如何精準觸發數位開關:真矽晶片上節點從 5V 漏到 1.8V
(非 0 非 1 模糊地帶)時,鄰近 NMOS 閘極進入半導通,產生劇烈類比暫態。
二值引擎的解法:為邊界節點定義閾值電壓 V_th,類比計時器算出跌破 V_th 時,
在數位側觸發 **Dirty Event** —— 強迫該節點在下半週期進 Settle Wave 重新評估。

## 7. 與 MOSSIM II 的對照(理論定位)

本引擎理論源頭同為 Bryant 開關級模型;MOSSIM II(Bryant, 1984)是通用 EDA 驗證
工具,本引擎是「已知晶片跑到極限速度」的特化實作。

**共享的靈魂**:事件驅動 + 導通群走訪、charge-sharing 電容決斷(浮接群由最大
電容節點 tie-break)、Gauss-Seidel 式迭代到穩態。

**MOSSIM II 多出的精緻度**:信號強度代數(Signal Strength Algebra)——
節點大小(電容)與電晶體強度(電導)各分多個離散等級,能算比例邏輯
(Ratioed Logic)的強弱抗衡與 pass-transistor 串聯衰減;狀態是 0/1/X 三值,
X 用於警告設計錯誤。

**本引擎的取捨(換取速度)**:二值化 + 絕對優先級(Gnd > Pwr > 外驅 > PullUp >
Floating 電容)、256-entry Flags-OR→LUT 的 O(1) 群組解析、零電阻理想開關、
SoA + unmanaged 指標的現代快取適配。原味 MOSSIM II 同機重寫會被數量級輾壓
(強度代數是熱迴圈裡昂貴的 ALU 運算;X 態會傳染、拖慢收斂;1980s 演算法
無 cache locality 概念)。

**MOSSIM II 對四類語意極限的理論表現**:

| 語意極限(ReportAC §3) | MOSSIM II 表現 |
|---|---|
| A1 比例式驅動打架(LAE/LXA) | ✅ 完美閃過 —— 強度代數正是為此發明 |
| A2 時窗重疊錯位(OpenBus err4 毛刺) | ❌ 照樣踩雷 —— 無奈秒/RC 延遲概念,1ns 毛刺與 100ns 驅動窗等量齊觀 |
| A3 透明閂鎖持續追隨(OpenBus err6) | ⚠️ 高度危險 —— 仍依賴 settle-到-靜止,無法模擬相位內連續追隨 |
| A4 管線延遲(DMA Abort) | ❌ 照樣踩雷 —— fixed-point 內訊號瞬間傳到底,無傳播延遲 |

結論:MOSSIM II 解決了「振幅被量化」的問題,沒解決「連續時間被量化」的問題。
四雷全避的唯一正解是 SPICE,但成本高出數百萬倍。因此
**「極速二值 switch-level 為底 + 特定時序盲區外掛行為層/時戳 shim」**
是在現代 CPU 上即時跑完 141 顆硬核測試的唯一理論最優解。

## 8. 模型適用性總結

- Switch-level 模型天生為**純邏輯**打造:理想開關(0Ω/∞)、零延遲、非黑即白。
  這是缺陷,更是效能護城河 —— 正因抹除了連續電壓與連續時間兩個維度,
  才能以 O(1) 查表推演 1.47 萬節點。
- 2A03/2C02 中 >99% 節點絕大多數時間表現如純數位;真正暴露類比特性的只有
  極少數邊界節點(浮接 Data Bus、DRAM 節點、中斷腳 RC)。
- 正確架構:「邏輯的歸邏輯,類比的歸類比」——
  99% 走現有 LUT 快路,1% 掛整數時戳計時器。

## 9. 落地路線圖(未排程,僅供未來評估)

1. **第一隻白老鼠**(二選一):
   - OAM DRAM 衰減(`oam_decay.nes` 反推 threshold);
   - CPU Open Bus 電容殘影(可能可取代現行 OB shim 的部分職責)。
2. 建 `analog_properties.json` Sidecar + 載入期預算 DelayCycles;
3. 以 Dirty Event 接回 settle wave(反向因果);
4. APU RC 濾波(IIR/FIR 外掛,不動熱迴圈);
5. 每步皆過:golden checksum 不變(類比層預設 OFF)、isolated ROM 驗證、
   interleaved-paired benchmark 證明效能損耗 ≤2%。

## 10. 與現行專案的銜接注意

- 現行 shim(OpenBus/DL/DmcAbort)= 此藍圖的**佔位符**;若時戳物理層落地,
  逐一評估可否被物理常數取代(而非兩層並存)。
- 「多邊形面積電容」與「正準排序」在另一份諮詢中列為**大爆炸候選**
  (checksum 必變、全域行為位移),進場順序必須在掛牌成績 bank 之後,
  且先用離線工具驗證,不直接動引擎。
- 類比層一切資料結構走「載入期靜態旗標 + 輔助陣列」,不得改動
  NodeStates/群組解析熱迴圈的既有記憶體布局。
