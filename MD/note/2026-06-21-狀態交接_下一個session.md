## 1. 專案一句話

把 Visual6502 風格的 switch-level netlist(2A03 + 2C02 整台 NES,NROM)轉成可分析/可驗證/可執行的邏輯模型。
**效能研究已結案,結論是一個有文件、可否證的負面結果**:event-driven switch-level 引擎(`src/AprVisual.S1/`)已是單核最快的黃金產物;所有抽象化(IR / AOT / GPU / Escape-1)都更慢;real-time(~42.95M hc/s)在這條路上不可達。
**現在的工作性質 = 在黃金引擎上累積「小而確定、bit-exact」的熱路徑微優化 + 維護對外的效能展示網站。** 不要重啟 GPU-throughput、IR、批次重算等已證死路(見記憶 dead-end 系列)。

## 4. /version/ 效能網站工作流(新基礎設施,務必先懂再動)

**對外網址**:`https://baxermux.org/myemu/AprVisual/version/`(FTPS 部署到 baxermux.org,**只寫 /version/,絕不碰 leaderboard 的 config.php / data**)。
**這跟 GitHub Pages(WebSite/)是兩個不同的站** —— /version/ 是 PHP host 上的逐版效能圖表;GitHub Pages 是文章站。

檔案都在 `tools/perf/`:

| 檔案 | 角色 |
|---|---|
| `metadata.csv` | 版本清單 + 日期 / commit / 標題 / 描述(每版一列;新版要在這加列) |
| `env_x64.json` / `env_arm64.json` | 該平台 CPU/環境 meta(Zen2 runs=8;arm boost=3.0G) |
| `prefetch_releases.ps1` | 把 GitHub release zip 抓到 `releases/<version>/` 解壓(idempotent;`-Force` 重抓) |
| `collect_x64.ps1` | 跑各 release 的 benchmark → `out/boost.csv` `sizes.csv` `ilhash.csv`(`-Locked` 另出 cyc/hc) |
| `collect_arm64.sh` | Pi 上從源碼逐 git-ref build + 跑 → `boost_arm.csv` `sizes_arm.csv` `ilhash_arm.csv` `temp_arm.csv`(溫度 >60°C 自動中止) |
| `ilhash/`(Program.cs + csproj) | **跨平台 IL 指紋小工具**;`dotnet ilhash.dll <AprVisual.S1.dll>` → 12-hex。對所有 `WireCore` 方法取 名稱+IL bytes 排序後 SHA256 |
| `build_json.py` | 合併上述 CSV + env → `web/<plat>/data.json`(schema v1);末端做 engine-group pass(用 `--ilhash` 以 `engine_il_hash` 分組);驗證全版 bit-exact 才寫出 |
| `deploy_version.ps1` | FTPS 上傳 `web/` 整包(index/app.js/app.css/api.php + 各平台 data.json);讀 `etc/ftp.txt`(gitignored,密碼絕不 echo) |
| `web/` | 前端:`index.html` `app.js` `app.css` `api.php`;`web/x64/data.json` `web/arm64/data.json` |
| `releases/` `out/` `raw/` | **gitignored**(機器本機快取;releases 約 1.1GB,19 版) |

**注意**:`web/x64/data.json` 被全域 `x64/` ignore 規則蓋掉,`.gitignore` 末有 negation 例外把它救回來(arm64 沒這問題)。兩平台 data.json 都有入版控。

**典型「發新版」流程**:
1. 發 GitHub `benchmark-<date>` release(用 `tools/release_benchmark.ps1`,見記憶 `benchmark-release-workflow`)。
2. `metadata.csv` 加列;`collect_arm64.sh` 的 ITEMS 加一行。
3. `pwsh tools/perf/prefetch_releases.ps1` → `pwsh tools/perf/collect_x64.ps1`(Zen2 鎖機跑)。
4. Pi 上跑 `collect_arm64.sh`。
5. `python tools/perf/build_json.py ... --ilhash out/ilhash.csv ...` 各平台一次(arm64 重用 x64 的 ilhash.csv,IL 平台無關)。
6. `pwsh tools/perf/deploy_version.ps1 -Platforms x64,arm64`。
7. commit + push。
8. 另外用 benchmark tool 跑最好一筆 → API 上傳排行榜 → 更新 WebSite/index.html + README flagship 數字。

**data.json 每版新增欄位**:`engine_il_hash`、`engine_group`(代表版 id)、`engine_rep`(bool)、`engine_dup`(bool)、`metrics.hc_s_best3_eff` / `hc_s_max_eff` / `realtime_x_eff`(圖表用組內最佳;raw 的 `hc_s_best3` 保留)。
