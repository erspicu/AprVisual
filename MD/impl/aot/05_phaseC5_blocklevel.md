## 1. 為什麼 partition "internal" ≠ AotEmitter "emittable"

```
Partition.Block #16 ppu.finex1:
  partitioner classification:
    - 188 "internal" nodes  ← non-pullup mids (anonymous pass-transistor nodes)
    - 27 "driven outputs"   ← pullup boundary nodes the block drives
    - 324 "boundary inputs" ← what the block reads

  AotEmitter coverage:
    - 7 / 188 internal emittable  ← only 7 mids fit a pull-up-based pattern
    - rest are dynamic latch storage — needs "latch" pattern (Phase C-7)
```

The mids are STORAGE nodes (no pull-up, value held by pass-transistor write). My pattern detection requires `Pullups > 0`, so they fall through to "unsupported".

**For Phase D 整 chip AOT**:Mid storage 不是問題 — 因為其他 block 不讀 internal mids,只讀 boundary outputs。**只要每個 block 的 boundary outputs 100% AOT-able 就夠**。Block #19/#22 都是這情況。

## 2. 實際 `.cs` 輸出範例(Block #19,APU 三角通道 bit 8)

```csharp
// Auto-generated by AotBlockBuilder.EmitSource
// Source partition block: #19 (cpu.tri_p8)
//   internal nodes : 24
//   driven outputs : 20
//   boundary inputs: 62
//   emittable      : 18 / 24 internal + 20 outputs
//   patterns       : unsupported=24

namespace AprVisual.Codegen.Generated
{
    public static unsafe class Block_19_cpu_tri_p8
    {
        public static void Eval(byte* nodeStates)
        {
            // mux_bus+pulldown[output]  cpu.sq1_envp0 (id 10532) ← inputs [11676,11206,11692,10547,11705,10637,10644]
            nodeStates[10532] = (byte)((nodeStates[11676] != 0) ? (byte)0 : (byte)(
                ((nodeStates[11206] == 0) ? 1 : nodeStates[11692]) &
                ((nodeStates[10547] == 0) ? 1 : nodeStates[11705]) &
                ((nodeStates[10637] == 0) ? 1 : nodeStates[10644])));
            // inverter+latch-write[output]  cpu.sq1_swpb0 (id 10780) ← inputs [12255]
            nodeStates[10780] = (byte)((nodeStates[12255] == 0) ? (byte)1 : (byte)0);
            // ... 16 more
        }
    }
}
```

**100% 純 boolean expressions**,沒有任何 transistor / BFS / group-resolution code。完全靜態編譯,可被任何 C# compiler 直接吃。

### 規模

- 5,402 bytes / 18 expressions
- 平均每 emittable 節點 ≈ 300 bytes 源碼(含 comment)
- 全 chip 推估:6,260 emittable × 300 = ~1.9 MB 純 C# 源碼 → ~30 個 partition .cs files

## 3. Pipeline 對齊 Gemini r3 §Q5 戰略

> 「**New Pipeline:**
>    Netlist → math-algos Partitioner → Macro-blocks Graph → C#/C++ Code Generator (AOT) → 編譯成一個沒有 BFS、只有純 Boolean Logic 與 bitwise operations 的全新模擬引擎。
>  放棄 S1 的 runtime:
>    既然 dispatcher 這麼快 (4.6% overhead),為什麼還要留著 S1?你可以把所有的 partition 都轉成 C 語言的 boolean equations,直接生成一整棵 C++ / C# 檔案。」
