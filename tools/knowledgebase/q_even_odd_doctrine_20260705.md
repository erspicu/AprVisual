# 諮詢:10-even_odd_timing 的「忠實偏差」定性是否恰當?

## 背景

我們維護一個電晶體級 NES 模擬器(Visual2A03 + Visual2C02 netlist,NTSC RP2A03G/RP2C02G,
NES-001 板卡),blargg/AccuracyCoin 測試目錄 145 項目前 142/3。其中
`ppu_vbl_nmi` 套件的第 10 項 **10-even_odd_timing** 被我們歸類為「永久忠實偏差」
(不修、公開文件說明),這次想請你檢驗這個定性與我們的解釋是否站得住腳。

## 測試內容

blargg `ppu_vbl_nmi/10-even_odd_timing`:NTSC 2C02 在**奇數幀**且背景渲染開啟時,
pre-render scanline 會跳過一個 dot(341→340)。此測試量測「跳點」相對 CPU 時脈的
落點與奇偶交替模式,對 CPU-PPU 上電相位對齊極度敏感。

## 我們的機器模型與量測(全部是 netlist 湧現行為,非設計參數)

1. **對齊空間**:CPU ÷12 除頻器自上電自由運轉;PPU ÷4 除頻器在 /res 釋放時重啟
   (兩者皆為探針實測的湧現行為)。相對相位因此只有 **4 種物理類**
   (我們編號 {1,7,5,3});用 reset-hold K=0..5 全掃描證明偶數 K 塌縮
   (K=2≡K=1、K=4≡K=3:判定、失敗碼、幀數逐項一致)—— 枚舉完備。
2. **互斥量測**:NMI-edge 家族(05-nmi_timing 等 8 項)在相位 {7,5} 通過;
   10-even_odd 在 {1,3} 通過;**完備枚舉下零交集** —— 在這個模型上,不存在
   一個上電狀態能同時滿足兩組。
3. **我們的選擇**:定格相位 7(K=1,即 blargg NMI-edge 測試校準的相位),
   8 項換 1 項;完整「對齊×判定矩陣」連同代價公開在報告頁卷宗。
4. **已承認的開放點**(公開 Q&A 文件裡寫明):相對相位空間已完整驗證,
   但**絕對 dot 位置是 netlist 湧現值,目前沒有獨立仲裁** —— 若我們的兩顆
   除頻器模型之間有 1-dot 的相對偏移誤差,互斥結論就會被推翻。
5. **對照組**:行為層模擬器 TriCNES(作者有真機)把上電偏移當參數調到全過,
   原始碼註解:「Shouldn't this be 0? I don't know why, but this passes all
   the tests if this is 7, so...?」—— 我們解讀為「對齊空間內的校準」,
   而非「存在單一物理上電狀態能全過」的證據。
6. **可驗證預測**(公開文件):在一台 NTSC NES-001 上,同一次上電(不關機)
   錄到 05-nmi_timing 與 10-even_odd_timing 都 PASS 的影像,即可推翻我們的
   互斥結論,並定位出除頻器相位模型的修正點。

## 請你回答(請以 NESdev 社群現有知識為準,不確定的請標明)

   全套在同一次上電通過?社群(NESdev wiki/論壇)對「NMI 時序測試組」與
   「even/odd 跳點測試」是否依同一個上電對齊變數、且可能互斥,有沒有
   現成討論或結論?

   (我們的模型量出 4 種;社群常說 "4 CPU-PPU clock alignments";
   另外還有 APU get/put 2 態。)我們「PPU ÷4 在 /res 釋放重啟、CPU ÷12
