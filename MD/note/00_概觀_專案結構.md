# MetalNES — 概觀與專案結構

## 1. 它是什麼

MetalNES 是一個**電晶體層級（transistor / switch-level）的 NES-001 模擬器**。README 自述：

> Transistor level NES-001 simulation. Builds on OSX only for now. No MMU support.
> Only possible due to the tremendous efforts of Visual2C02 and Visual2A03.
> Added support chips for main board. Support voltage ladders for composite output and audio.

關鍵特徵：

- **不是行為級模擬器**。它不寫「6502 的 LDA 指令做什麼」，而是把 2A03 / 2C02 的**實際電晶體網表**（來自 Visual2A03 / Visual2C02 逆向）載進來，逐 half-cycle 推進電路狀態。
- **整台機板都模擬**。不只兩顆自訂晶片，連 NES-001 主機板上的標準邏輯晶片都建了模型：
  - `74LS139` ×1（u3，位址解碼器）
  - `74LS373` ×1（u2，PPU 位址 latch）
  - `74LS368` ×2（u7/u8，控制器讀取的三態緩衝）
  - `74HC04` ×1（u9，反相器，做 PPU A13 反相）
  - `SRAM2K` ×2（u1 = CPU work RAM、u4 = PPU VRAM）
  - `4021` ×2（在 `nes-pad` 模組裡，控制器的 8 位元並轉串移位暫存器）
  - `nes-cic1`（u10，CIC 防盜晶片，這裡只用它的 reset 邏輯）
- **真實 reset 行為**：拉高 `res` → 跑 192 個 half-cycle → 拉低 `res`，讓晶片自己跑 reset sequence。**不是**設計文件警告的「假裝 bus 讀到 NOP」。
- **輸出真實類比訊號形態**：composite video 走電壓階梯（voltage ladder），audio 走 R-2R 之類的 DAC ladder，再由 host 端做 NTSC shader / 重採樣。
- **可跑 nesdev 測試 ROM**：`data/roms/nes-test-roms/` 裡裝了 blargg 的 CPU/PPU/APU 測試套件，模擬器會自動偵測 `$6000` 的結果簽章並把結果印出來、存圖、存音訊。

## 2. 技術棧

| 面向 | 用什麼 |
|------|--------|
| 語言 | C++17 |
| 建構 | CMake（`CMakeLists.txt`） |
| GUI | Dear ImGui（`external/imgui`） |
| 渲染後端 | D3D11 / D3D9 / OpenGL ES / Metal / null（`source/render/`，跨平台抽象） |
| 平台殼 | OSX（`app/OSX`，Metal）、Linux（`app/Linux`）、Emscripten/WASM（`app/emscripten`） |
| 三角化 | `external/triangle`（用來把晶片 polygon 轉成可畫的三角形，做晶片視覺化） |
| Shader | HLSL，用 `hlsl2glslfork` 轉到 GLSL |
| 序列化 | 自家 JSON（`Core/JsonParser.h`、`serializer.cpp`） |
| 模擬執行緒 | `std::thread`（`wire_thread.h` + `dispatch_queue.h`） |

> ⚠️ 註：這份原始碼是 OSX 為主，但模擬核心（`source/metalnes/wire_*`、`system.cpp`、`handler_*`）是純 C++，與平台無關。我們要參考的就是這塊，平台殼與渲染都可忽略。

## 4. 與本專案（AprVisual）的關係

| 我們的設計文件主張 | MetalNES 的狀態 |
|--------------------|-----------------|
| 解析 Visual2A03 / 2C02 網表成 graph | ✅ 完整做了（`wire_defs` + `Wires::addInstance`） |
| 處理 pull-up（`PullType == '+'`） | ✅ `node::pullups`，segdef `+` 與 module 的 `pullups:[]` 都吃 |
| 偵測並斬斷迴路 | ⚠️ **沒有顯式做**。靠 `processQueue` 的迭代收斂 +「最大電容節點 hold」啟發式繞過（見 01 號文件） |
| 把 dynamic node 抽象成 `current_state` | ⚠️ 隱式：浮接節點靠 `node_state` flag 保留前值；group 全浮接時用「連線最多的節點 wins」 |
| Bus 整合（read addr / R/W / drive data） | ✅ 用 handler + callback 機制（RAM/ROM watch 觸發節點） |
| Reset vector / 真實 reset | ✅ 真做了（`setHigh(res); step(192); setLow(res)`） |
| 一個 graph 當 single source of truth，emit CPU / CUDA / Verilog | ❌ **沒有**。它只有 CPU 直譯，沒有 IR、沒有 codegen、沒有 CUDA |
| Bit-parallel / SoA / GPU 大規模平行 | ❌ 沒有。單執行緒、一台 NES |
| 全封裝晶片架構（ROM/RAM/CPU/PPU 全在一起） | ✅ 概念上就是這個（只是在 CPU 上、不在 GPU） |

**結論**：MetalNES 把我們設計文件 Phase 0–2（解析、圖、CPU evaluator、bus 整合）+ Phase 5（2A03 + 卡匣）+ 一部分 Phase 6（PPU、整機）全做完了，而且做得很完整、能跑測試 ROM。它**沒做**的是 Phase 3–4 + Phase 6 的 GPU 部分（IR、Verilog、CUDA、bit-slicing、大規模平行）—— 而那正是我們設計文件主張的「真正有價值的中間層」。所以：

> MetalNES = 我們的 CPU evaluator 參考實作 + 我們的「不要再做一次的部分」+ 我們的「往前推進的起點」。
