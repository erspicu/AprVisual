# Explicit/Implicit DMA Abort 戰役:開場取證(相位問題)

日期:2026-07-14 · 對象:`$479` ExplicitDMAAbort / `$478` ImplicitDMAAbort(皆 run4 `$0A` = FAIL err2)

## 1. 測試機制(全破解)

- err2 = **答案鍵比對**:16 個 X(0-15)掃相位,量測「DMA 中止編排後下一次 DMA 的相位」寫入 `$50-$5F`,
  與硬體答案鍵 `04 04 04 04 04 04 03 04 01 01 00 00 00 00 00 00` 逐位比對。
- **量測 = open-bus DMA 碰撞偵測器**(`CalculateDMADuration`):迴圈 `LDA $4000`(open bus,平常讀
  `$40`=自己的運算元高位),每圈滑 560 週期讓讀取點相對 DMA(576 週期/位元組)每圈提前;撞上 DMA
  那圈讀到 **DMC 樣本位元組 `$00`** → BEQ 跳出,Y=圈數=相位。
- X 掃相位:`Clockslide64_Minus_A`,X 越大 disable(`STA $4015`=0)越早,掃過目標 DMA 的 4 週期窗。
- 速率:`$4E` = rate idx $E = **72 週期/位元 → 576 週期/位元組**(idx $F 才是 54/432)。

## 2. S1 量測結果與硬體差異

S1 全 X 讀 **0**;硬體 `4,4,4,4,4,4,3,4,1,1,0,0,0,0,0,0`。
**X≥10(完全中止段)S1 與硬體一致**;分歧只在 X=0-9(未中止/部分中止段)。

## 3. 微架構顯微鏡(rdy + pcm 節點,全部合理)

工具:rdy 停機時間線 + pcm raw-id 顯微鏡(`cpu.#14059`=pcm_dma_active、`#11094`=loadbuf、
`#11093`=loadsr、`#11102`=shiftsr、具名 `pcm_en`/`pcm_lc[11:0]`)。X=0 迭代逐拍:

```
15874307 loadsr(SR 取走 buffer → buffer 空)
15874379 dma+loadbuf(目標 DMA 重抓,buffer 滿)     ← stall 84t ≈ 3.5 cyc
15874559 en 1→0(disable —— 晚了 ~7.5 cyc,X=0 本來就該 miss ✓)
  en=0 期間:shiftsr 照每 1728t(72 cyc)走 ✓(硬體輸出單元也不停)
15888131 loadsr(SR 又取走 buffer —— en=0 也照搬 ✓ 硬體同)→ buffer 空
15888263 en 0→1(re-enable)
15888443 dma+loadbuf(立即重抓 —— buffer 空,合法 ✓)
```

**每一步單獨看都符合硬體語意。** 分歧在整體相位:S1 的 re-enable 後重抓 = 目標 DMA + **586** 週期
(自然槽位 576 + ~10 抖動);答案鍵 X=0-9 段吃的就是這 ±10 週期。

CheckDMATiming(Y=4 閘)通過 → 基本 load-DMA 時序與硬體一致;DMASync 碰撞輪詢正常
(stall @ab=$4000 可直接當同步標記)。
