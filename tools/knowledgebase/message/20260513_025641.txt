**Game Boy 開機畫面那個掉下來的「Nintendo」Logo，是一個埋在矽晶片裡的商標法陷阱。**

你記得小時候打開 Game Boy 時，螢幕上方會緩緩掉下一個「Nintendo」的字樣，然後「叮」一聲，遊戲才開始對吧？（如果有灰塵接觸不良，那個字會變成一塊破圖的黑色方塊，然後遊戲就當在那裡）。

這個機制的運作原理是：Game Boy 主機內部有一顆極小的 256-byte Boot ROM。當你開機時，這顆 ROM 會去讀取卡匣標頭（header）裡特定位址（0x0104 - 0x0133）的 48 bytes 資料。它會把這段資料**畫在螢幕上**（這就是為什麼即使卡匣壞了，字還是會掉下來），然後，Boot ROM 會把卡匣裡的這 48 bytes，跟主機內部自己存的一份 Nintendo Logo 進行**位元對位元（bit-by-bit）的比對**。

只有完全吻合，Boot ROM 才會把控制權交給卡匣。只要差一個 bit，Game Boy 就會死機。

但最 delightful 的部分不是這個技術機制，而是它的**動機**。

任天堂當時深受未授權的盜版或第三方遊戲廠所苦。如果他們單純做一個「密碼鎖」（例如卡匣要送出 0x55 0xAA 才能開機），第三方廠商可以直接把這串密碼逆向工程寫進去，這在當時的法律上通常會被視為「為了達到機器相容性的合理使用（Fair Use）」，任天堂告不贏。

所以任天堂的工程師跟法務想到了一個天才（且邪惡）的點子：**他們把「密碼」設成了任天堂自己註冊的商標（Trademark）圖像。**

這意味著：如果你是一家沒有拿到任天堂授權的第三方廠商，你想讓你的卡匣在 Game Boy 上跑起來，你在技術上**被迫**得把那 48 bytes 的 Nintendo Logo 寫進你的程式碼裡。
而一旦你這麼做了，玩家把你的非授權卡匣插進 Game Boy 開機時，螢幕上就會大剌剌地顯示「Nintendo」！

這在法律上構成了完美的**商標侵權（Trademark Infringement）與虛假關聯**。你不能在法庭上說「我只是在傳送一段相容性資料」，因為任天堂的律師會說：「法官大人，你看，他的遊戲一開機就打著我們家的 Logo 招搖撞騙！」
