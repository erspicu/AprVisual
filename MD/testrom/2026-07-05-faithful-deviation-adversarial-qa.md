# 忠實偏差對抗性攻防 —— 假設一位資深模擬器開發者來質疑

> 這份文件反其道而行:我們**先替最強的質疑者把攻擊寫好**,再逐條應答。
> 對象:兩個永久忠實偏差(`10-even_odd_timing`、`cpu_dummy_writes_oam`)
> 以及「忠實偏差」這個分類本身。English version: `2026-07-05-faithful-deviation-adversarial-qa.en.md`。
> 所有主張的證據鏈見[知識庫](00_測試修復知識庫_總綱.md)與報告頁卷宗。

## 0. 我們的判定標準(先立規則,再談個案)

一個 FAIL 要列入「忠實偏差」,必須同時滿足:

1. **測試作者自己的文件**記載該行為依硬體而異(readme / 原始碼註解 / 畫面自述)
2. **社群文獻佐證**(NESdev wiki / 論壇)
3. 可行時,**獨立模型交叉驗證**(例:emu-russia 對同一晶粒的獨立逆向)
4. **凡真機行為確定的,我們一律修**——本專案已修 10+ 項,含三個類比賽跑 shim;
   「忠實偏差」不是垃圾桶,是用盡修復手段後的殘餘
5. 每條偏差**附可否證的預測**——歡迎用真機資料推翻我們

## Case 1:`10-even_odd_timing`

### Q1|「我的真 NES 有時能全過 ppu_vbl_nmi 十項。你們的『互斥』是你們模擬器的假影,不是物理。」

**A:** 我們的主張比你以為的窄,而且刻意如此:

- 我們主張的是:**在這個矽晶模型上**,可達的上電對齊空間(完備枚舉,見下)中,
  沒有任何單一狀態同時滿足 NMI-edge 家族(8 項)與 10-even_odd。
- 完備性不是抽樣:K=0..5 掃描證明中間 reset 偏移量化塌縮回同樣 4 類
  (K=2≡K=1、K=4≡K=3;判定、失敗碼、幀數逐項一致)——除頻器對以整
  clk0 週期重啟,**只存在這 4 個相對相位**。
- 「PPU ÷4 在 /res 釋放時重啟、CPU ÷12 自由運轉」不是我們的設計選擇,
  是 netlist 探針量出來的**湧現行為**。
- 你的真機觀察(若成立)與我們的主張並不直接矛盾——它會指向一個
  **可定位的具體錯誤**:我們兩顆除頻器模型之間 1 dot 的相對偏移。
  這正是我們寫在卷宗裡的開放問題。

**可否證預測:** 找一台 NTSC NES-001,錄下**同一次上電**(不重開機)下
`05-nmi_timing` 與 `10-even_odd_timing` 都 PASS 的影像證據。做到,我們的
互斥主張即被推翻,並可據此校正除頻器相位模型——我們會把它當成禮物。

### Q2|「你們選對齊 7 是為了刷分——挑一個輸最少的。」

**A:** 部分正確,而且我們就是這樣寫的:任何**固定**對齊系統(包括一台不重
開機的真 NES)都要在兩組測試之間取捨。我們選 blargg 的 NMI-edge 測試
校準的那個相位(8 項對 1 項),並把**完整的對齊×判定矩陣**連同代價
(10-even_odd FAIL)公開在卷宗裡。刷分的做法是隱藏矩陣;我們把矩陣
當成主要成果。

### Q3|「行為層模擬器全過是因為共識時序模型允許。netlist 過不了,更可能是你們的模擬有 1-dot 錯——你們驗過 VBL 設定點和 skip 點的絕對位置嗎?」

**A:** 誠實回答:**相對相位空間已完備驗證;絕對 dot 位置是 netlist 湧現值,
尚無獨立仲裁**。這是本卷宗唯一承認的裸露面。緩解與後續:

- 對照組:TriCNES(作者有真機)全過十項的方法是**手調上電偏移**,其原始碼
  註解原話:「Shouldn't this be 0? I don't know why, but this passes all the
  tests if this is 7, so...?」——那是對齊空間裡的校準行為,不能當成
  「單一物理狀態能全過」的證據。
- 已排入的仲裁路徑:emu-russia **PPUSim**(同一 2C02 晶粒的獨立電路逆向)
  可跑同場景;我們在 DMC 案例已用其 APUSim 做過同型仲裁(結論:兩個獨立
  矽晶模型同敗 = 抽象極限,不是誰的 bug)。

## Case 2:`cpu_dummy_writes_oam`

### Q1|「這測試的主體是 CPU dummy write(RMW 指令的兩次寫入),OAM 只是驗證載具。你們的 FAIL 很可能是在遮掩一個真的 CPU bug。」

**A:** 這是最強的質疑,也有最硬的反證:**姊妹測試
`cpu_dummy_writes_ppumem` 用完全相同的 dummy-write 機制、改用 PPU 記憶體
當驗證載具——我們 PASS**。CPU 的兩次寫入行為被獨立驗證過了。
`_oam` 變體失敗的位置,正好是它自己聲明不可靠的那個載具。

### Q2|「AprNes 和 TriCNES 都通過這兩個測試。行為層做得出來,你們做不出來?」

**A:** 這一題的答案是**機型版次**,而且有 NESdev 權威文句(2026 現行版):

> "Writing to OAMADDR on the **2C02G** causes OAM corruption —
> this usually seems to **copy sprites 8 and 9 (address $20) over the
> 8-byte row at the target address**."
> "It is known that in the **2C03, 2C04, 2C05, and 2C07, OAMADDR works
> as intended**."

- 我們的基準機型 **RP2C02G 正是有這顆 bug 的版次**;後期 PPU 把它修掉了。
- blargg 的測試都要寫 $2003(OAMADDR)來定位讀取——在真 G 主機上
  就會踩進損毀;他畫面上那句「emulators 常過、真 NES 不過」與此吻合。
- AprNes / TriCNES 通過,是因為它們實作的是**理想化(= 修正版次)的
  OAMADDR**;Mesen2 把 G 的行為做成**預設關閉**的選配,名字就叫
  `EnablePpuOamRowCorruption`(row 損毀——與 NESdev 的 8-byte row
  敘述一致),開啟後同樣會失敗。
- 我們的 OAM 與其定址邏輯是晶粒描繪的實體電路,損毀行為**湧現**而非
  實作選項——要「過」就得把 G 的矽晶行為換成修正版次的理想陣列,
  那是讓模擬變得更不忠實。

### Q3|「引用作者警告是免死金牌。」

**A:** 引畫面原文(截圖可查,任何人可重跑):

> "Requirement: OAM memory reads MUST be reliable. This is often the case
> on emulators, but **NOT on the real NES**."

作者把「模擬器常過、真機不過」印在測試開頭的畫面上;配上 Q2 的 NESdev
機制文句(2005 年作者觀察到的「不可靠」,2026 年的社群已定位為 G 版次的
OAMADDR 寫入損毀),警告與機制互相咬合,不是修辭。配套證據:同作者
`oam_read` 的 readme 記載**同一台真機**不同次上電出現四種圖樣、三種以
Failed 收場——我們的判定落在這個抽籤家族內(目前恰落在通過圖樣,
「會變」本身就是忠實行為)。

### Q4|「你們的損毀圖樣未必和真 G 的『sprite 8/9 列複製』逐位元相同——可能是第三種人造態。」

**A:** 部分成立,照單全收:我們尚未逐位元驗證損毀圖樣與 NESdev 敘述的
對應(cell 也不建模電荷衰減——已文件化缺口)。我們主張的是**類別忠實**:
G 版次「寫 OAMADDR → OAM 不可靠」的行為類別,而非特定位元圖樣
(作者自述「這些值大概是我這台 NES 獨有的」——精確圖樣本來就依機而異)。
逐位元對照(netlist 損毀 vs sprite-8/9-row 敘述)列為開放工作。

**可否證預測(機型版):** 在真 **RP2C02G** 主機上,`cpu_dummy_writes_oam`
失敗、`oam_read` 呈四圖樣抽籤;在 **2C03/2C05(RGB PPU)** 主機上兩者
應通過。有 Vs./RGB 改機的玩家可以直接檢驗這條——結果任一方向都歡迎。

## Meta|「『忠實偏差』整個分類就是無法否證的遁詞——任何 bug 都能宣稱『這是類比』。」

**A:** 用行為記錄反駁:

1. **我們修掉的比我們豁免的多一個數量級**:計分 115/26 → 140/5,
   已修 10+ 項,其中三項(DMC latch、ALU latch、frame-IRQ)本身就是
   「類比賽跑」——真機行為確定,我們就修,不豁免。
2. 分類有**入場標準**(§0 五條),每條偏差附**可否證預測**。
3. 有獨立模型仲裁的先例:DMC 案例我們跑了 emu-russia APUSim,
   兩個獨立矽晶模型以同樣方式失敗、行為層模型通過——
   這是「抽象極限」的可重複實驗證據,不是修辭。
4. 分類是**雙向的**:oam_read 曾列忠實偏差(失敗家族),引擎修正後
   落到通過圖樣,我們照樣把它移出失敗清單並改寫條目——
   分類跟著證據走,不跟著分數走。

## 結語

這兩條偏差的共同結構:**測試把「作者那台機器的特定物理」編碼成了期望值**。
行為層模擬器實作共識規則,自然全過;電晶體級模擬重現的是一顆特定晶片的
物理,於是在「物理本來就依機而異」的地方誠實地分歧。我們能提供、而全過
的模擬器不能提供的,是每一個分歧點的**電路級解釋與可否證的邊界**。
