## 0. S1 是什麼

把 Visual6502-style 的 **switch-level transistor netlist**（2A03 NES CPU + 2C02 PPU + 板級 TTL，`segdefs`/`transdefs`/`nodenames`）讀進來、**逐電晶體模擬**整顆 NES、跑 blargg 測試會過。它是 MetalNES（C++）的 C# 重寫，而 MetalNES 核心又是 Visual6502 `chipsim.js` 的最佳化 port。

**它不模擬邏輯閘 —— 它模擬電晶體和導線。** 一顆 NAND gate 在這裡是 4 顆 NMOS 電晶體 + 連線；CPU 的一個 register bit 是一群交叉耦合的電晶體。沒有「指令解碼」「ALU」這種抽象 —— 只有 ~14,700 個 node（導線網）和 ~26,800 顆電晶體（開關）。

效能：**~47K master half-cycle / 秒**（~14s/幀、~0.07 FPS）。離 real-time（~16ms/幀）差 ~840×。**S1 的目的不是即時、是「正確 + 可作為 golden reference」** —— 它是後續 S2/S3/S4（IR 萃取、codegen）逐 node 對照的基準。

## 1. switch-level 模型

**三個東西**：
- **node**（導線網）—— 一根電氣上連通的金屬/多晶矽。有一個 0/1 狀態。`vcc`(id=1) / `vss`(id=2) 是兩個特殊 node。
- **transistor**（NMOS 開關）—— `(gate, c1, c2)`。當 `gate` node 為 1，c1↔c2 *導通*（變成同一個電氣節點）；gate 為 0，斷開。c1/c2 可互換。有個 `weak` flag（depletion-load，弱上拉）。
- **group**（導通群）—— 透過*目前導通的*電晶體連在一起的一群 node。它們在物理上是同一個電氣節點、必須同值。

**怎麼決定一個 node 的值（group resolution）**：從該 node 出發、沿*導通的* channel 走出整個導通群、把群裡每個 member 的 flags OR 起來，然後依**物理優先序**決議：

```
1. ForceCompute 特例：若群同時有 Gnd 和 Pwr → 兩者抵銷（某些 bus node）
2. Gnd     → 0   (接地獲勝)
3. Pwr     → 1   (VCC)
4. SetHigh → 1   (外部驅動高，如 pin)
5. SetLow  → 0   (外部驅動低)
6. PullUp  → 1   (上拉電阻 / 弱電晶體)
7. State   → 1   (上一刻是高 → 寄生電容保持)
8. else    → 0
+ 若 OR 出來的 flags 全空（純浮接群）→ 取群裡「電容最大」(連線數最多) node 的當前值
```

這個優先序 = NMOS 電路的物理（強驅動贏過弱、有驅動贏過浮接保持）。

## 2. 資料結構（SoA + unmanaged hot path）

熱資料全是 unmanaged pointer（`NativeMemory.AlignedAlloc`），SoA 排列、zero bounds-check、steady-state 零配置：

```csharp
// WireCore.cs — hot per-node data, allocated by Reset() once node count is known
public static byte*     NodeStates;   // [nodeId] = 0/1 — read constantly during the BFS
public static NodeInfo* NodeInfos;    // [nodeId] = flags + tlist indices (struct below)
public static byte*     FlagsToState; // [256] — precomputed group-resolution LUT
public static int*      TransistorList; // one big int[] of null(0)-terminated sub-lists (cache-friendly)

// double-buffered recalc queue (the dirty-set)
public static int* RecalcList;  public static int* RecalcListNext;
public static int* RecalcHash;  public static int* RecalcHashNext;   // dedupe: nonzero = already queued
public static int  RecalcListCount, RecalcListNextCount;

// Per-node hot record — kept ~32 bytes so the array stays cache-dense.
internal struct NodeInfo
{
    public NodeFlags Flags;
    public int Connections;   // c1c2s+gates count — "capacitance" proxy for the floating tie-break
    public int TlistGates;    // index into TransistorList: (c1,c2, c1,c2, ..., 0) — transistors this node GATES
    public int TlistC1c2s;    // (gate,other, gate,other, ..., 0) — channels to a normal node
    public int TlistC1gnd;    // (gate, gate, ..., 0) — channels whose far end is GND
    public int TlistC1pwr;    // (gate, gate, ..., 0) — channels whose far end is VCC
}

[Flags] public enum NodeFlags : byte {
    None=0, State=1<<0, PullUp=1<<1, SetHigh=1<<2, SetLow=1<<3,
    Pwr=1<<4, Gnd=1<<5, ForceCompute=1<<6, HasCallback=1<<7,
}
```

**關鍵設計**：每個 node 的電晶體被預先*分桶*成四條 sub-list（gates / 普通-channel / GND-channel / VCC-channel），各自是 `TransistorList` 裡一段 0-terminated 的連續 int —— 走訪時順序存取、cache-friendly。

## 3. 核心演算法：event-driven settle（一個半週期）

S1 是 **event-driven** 的：每半週期*只*重算「輸入可能變了」的 node（dirty-set），不是全部重算。

```csharp
// WireCore.Recalc.cs — one half-cycle
private static void StepCycle()
{
    RunHandlerChain();   // clk toggle (SetHigh/SetLow on "clk") + memory/video callbacks → enqueue + settle
    if (TraceLevel != 0) CaptureTraceLine();
    Time++;
}
```

clk handler 對 `clk` node 做 `SetHigh`/`SetLow` → `RecalcNodeList` → `EnqueueNode` + `ProcessQueue`。`ProcessQueue` 是 BFS 一波波 settle 到 quiescent：

```csharp
private static void ProcessQueue()
{
    int iteration = 0;
    while (RecalcListNextCount != 0)
    {
        if (++iteration > MaxSettlePasses) { /* abort, clear, warn */ break; }
        // swap "next" ↔ "current" (double-buffered)
        (RecalcList, RecalcListNext) = (RecalcListNext, RecalcList);
        (RecalcHash, RecalcHashNext) = (RecalcHashNext, RecalcHash);
        RecalcListCount = RecalcListNextCount; RecalcListNextCount = 0;
        for (int i = 0; i < RecalcListCount; i++)
        {
            int nn = RecalcList[i];
            if (RecalcHash[nn] != 0) { RecalcHash[nn] = 0; RecalcNode(nn); }  // skip if absorbed into a group already
        }
        RecalcListCount = 0;
    }
    InvokeCallbacks();   // memory writes / video frame writes, once the dust settles
}

internal static void RecalcNode(int nn)
{
    if (nn == Npwr || nn == Ngnd) return;
    byte newState = ComputeNodeGroup(nn);                       // walk the conducting group, resolve
    for (int i = 0; i < _groupCount; i++) SetNodeState(_groupBuf[i], newState);  // set ALL group members
    if ((_groupFlags & NodeFlags.HasCallback) != 0)             // any watched node in the group?
        for (int i = 0; i < _groupCount; i++) { var node = Nodes[_groupBuf[i]]; if (node?.Callback != null) EnqueueCallback(node.Callback); }
}
```

群走訪 + 解析（the heart），在 `WireCore.Group.cs`：

```csharp
private static byte ComputeNodeGroup(int nn)
{
    for (int i = 0; i < _groupCount; i++) _inGroup[_groupBuf[i]] = 0;   // clear last group's dedup flags
    _groupFlags = NodeFlags.None; _groupCount = 0; _maxState = 0; _maxConnections = 0;
    AddNodeToGroup(nn);
    return GetNodeValue();
}

private static void AddNodeToGroup(int nn)   // recursive DFS over ON transistors
{
    if (_inGroup[nn] != 0) return;            // O(1) dedup (← improvement over MetalNES's linear scan)
    _inGroup[nn] = 1;
    ref NodeInfo ns = ref NodeInfos[nn];
    _groupBuf[_groupCount++] = nn;
    if (ns.Connections > _maxConnections) { _maxState = NodeStates[nn]; _maxConnections = ns.Connections; }  // floating tie-break
    RecalcHash[nn] = 0;                        // absorbed into this group; won't need separate re-processing
    _groupFlags |= ns.Flags;
    if (ns.TlistC1c2s != 0)                    // channels to normal nodes: (gate, other) pairs
    {
        int* p = TransistorList + ns.TlistC1c2s;
        while (*p != 0) { int gate = *p++; int other = *p++; if (NodeStates[gate] != 0) AddNodeToGroup(other); }
    }
    if (ns.TlistC1gnd != 0) { int* p = TransistorList + ns.TlistC1gnd; while (*p != 0) { int gate = *p++; if (NodeStates[gate] != 0) { _groupFlags |= NodeFlags.Gnd; break; } } }
    if (ns.TlistC1pwr != 0) { int* p = TransistorList + ns.TlistC1pwr; while (*p != 0) { int gate = *p++; if (NodeStates[gate] != 0) { _groupFlags |= NodeFlags.Pwr; break; } } }
}

private static byte GetNodeValue()
{
    if ((_groupFlags & NodeFlags.ForceCompute) != 0 && (_groupFlags & NodeFlags.Gnd) != 0 && (_groupFlags & NodeFlags.Pwr) != 0)
        { _groupFlags &= ~NodeFlags.Gnd; _groupFlags &= ~NodeFlags.Pwr; }
    if (_groupFlags == NodeFlags.None) return _maxState;        // purely floating: largest-cap node wins
    return FlagsToState[(int)_groupFlags];                      // else: one LUT lookup
}
```

值寫回 + 傳播（`WireCore.Recalc.cs`）：

```csharp
private static void SetNodeState(int nn, byte newState)
{
    if (NodeStates[nn] == newState) return;       // ← EARLY RETURN: 值沒變、不傳播（最關鍵的剪枝）
    NodeStates[nn] = newState;
    ref NodeInfo ns = ref NodeInfos[nn];
    if (ns.TlistGates != 0)                       // re-examine transistors this node GATES
    {
        int* p = TransistorList + ns.TlistGates;
        while (*p != 0) {
            int c1 = *p++; int c2 = *p++;
            EnqueueNode(c1);
            if (newState == 0 && c2 != Npwr && c2 != Ngnd) EnqueueNode(c2);  // gate低→可能斷開→遠端要重算
        }
    }
}

private static void EnqueueNode(int nn)
{
    if (nn == Npwr || nn == Ngnd) return;
    if (RecalcHashNext[nn] == 0) { RecalcListNext[RecalcListNextCount++] = nn; RecalcHashNext[nn] = 1; }  // dedup
}
```

**注意 S1 怎麼傳值**：值*不是*靠 queue 沿 channel 傳的，是靠 group walk —— `RecalcNode(c1)` 的 `ComputeNodeGroup` 走過整個導通群、把*所有* member 一次設成群值。queue 裡 enqueue channel 端點是為了「gate 變了 → 導通拓樸變了 → 端點的群要重新解析」。這個區分是後面 #1 加速的關鍵。

## 4. 讓 S1 快的五個機制

S1 ~47K hc/s。快的原因是這五個（前四個 inherited from MetalNES/chipsim，第五個是實作）：

1. **Event-driven dirty-set**（`RecalcList`/`ProcessQueue`）—— 每半週期*只*重算變動的 ~幾百個 node（D ≈ 600），不是全部 ~14,700。這是 **O(D) 不是 O(N)**、最大的算法勝。
2. **兩個 dirty-flag（O(1) dedup）** —— `RecalcHash`（佇列去重：`if (RecalcHashNext[nn]==0)`）+ `_inGroup`（群走訪去重：`if (_inGroup[nn]!=0) return`）。把「已在佇列/已在群」的重複工砍掉。
3. **三個 early-return**：
   - `SetNodeState`：`if (NodeStates[nn]==newState) return;` —— **最關鍵** —— 值沒變就*不傳播*、dirty-set 只在「值真的變了」的地方擴張、不雪崩。
   - `ProcessQueue`：`if (RecalcHash[nn]!=0)` —— 已被群走訪吸收的 node 跳過。
   - `AddNodeToGroup`：`if (_inGroup[nn]!=0) return;` —— 群裡已訪過的不重訪。
4. **256-entry `FlagsToState` LUT** —— 群解析（GND>VCC>...>hold 那串優先序）變成 branch-free 一次查表。LUT 256 bytes、永遠在 L1。建表：

```csharp
// WireCore.Group.cs — build the 256-entry LUT once (FlagsToStateOf is the pure resolution fn)
public static void BuildFlagsToStateTable()
{ for (int i = 0; i < 256; i++) FlagsToState[i] = FlagsToStateOf((NodeFlags)i) ? (byte)1 : (byte)0; }

private static bool FlagsToStateOf(NodeFlags f)
{
    if ((f & NodeFlags.ForceCompute)!=0 && (f & NodeFlags.Gnd)!=0 && (f & NodeFlags.Pwr)!=0) { f &= ~NodeFlags.Gnd; f &= ~NodeFlags.Pwr; }
    if ((f & NodeFlags.Gnd)!=0) return false;   if ((f & NodeFlags.Pwr)!=0) return true;
    if ((f & NodeFlags.SetHigh)!=0) return true; if ((f & NodeFlags.SetLow)!=0) return false;
    if ((f & NodeFlags.PullUp)!=0) return true;  if ((f & NodeFlags.State)!=0) return true;
    return false;
}
```

5. **Unmanaged hot path** —— `byte* NodeStates` / `int* TransistorList` / `NodeInfo* NodeInfos` / 各 `int*` 佇列（`NativeMemory.AlignedAlloc`）、zero bounds-check 內迴圈、steady-state 零配置、no GC、no string。讓每個 group walk / SetNodeState 是 ~十幾個 CPU cycle（~3-5ns）。

## 5. S1.5 lowering（`WireCore.Lower.cs`）

讀進來的 netlist 先做一個 *behaviour-preserving* 的壓縮，再給 `Reset()` / 後續 S2：

- **static-group merge** —— 只被 always-on 非弱電晶體（gate==vcc）連在一起的兩個 node *永遠*同值 → union-find 折成同一個 node。
- **dead transistor 移除** —— gate==vss 永遠不導通 → 丟掉。
- **compaction** —— 倖存者重編成 dense 0..N（0/1/2 保留）、丟 self-loop、re-dedup `(gate,c1,c2)`。

實測：node `15164 → 14723`（merge 441）、transistor `27305 → 26775`（drop 530）、~+5% 速度。`--no-lower` 可關掉做 A/B。

## 6. math-algos 加速實驗（G / Y / X / #1）

在 S1 完成基礎上、不重做 S2/IR/codegen，純找演算法加速。設計 + Gemini 評估在 `math-algos/00_design.md`、完整實測在 `math-algos/01_results.md`。摘要 + 關鍵 code：

### G —— Reverse Cuthill-McKee node-id 重排（`--rcm`）→ ~1.04×（幾乎沒用）
把 node id 依「transistor 鄰接」重排、讓常一起 access 的 node 在記憶體相鄰。**結果：~1.04×（boot）/ 打平（穩態）。** 原因：S1 的 `NodeStates`（~15KB）已塞進 L1d、**cache 從來不是瓶頸**、重排沒得救。`src/AprVisual/Sim/WireCore.NodeOrdering.cs`。

### Y —— SIMD-unrolled inner walk（`--simd-queue`）→ ~1.00×（中性）
把 `AddNodeToGroup` 的 channel walk unroll-4、4 個 `NodeStates[gate]` byte-load 排在 branch 前、暴露 memory-level parallelism（真 AVX2 gather 在 C# 對 `byte*` 沒支援）。**結果：打平。** 大多 node transistor list ≤5、unroll 進不去 fast path；且同樣不打 D。`src/AprVisual/Sim/WireCore.GroupSimd.cs`。

### X —— Oblivious all-sweep（`--oblivious`）→ **121× 倒退**
放棄 dirty-set、每半週期*無條件*重算所有 node 到 fixpoint。**結果：~2,897 µs/hc vs S1 ~24 µs/hc = 121× 慢。** 證明 dirty-set 的 O(D) 是 essential、換 O(N) 是災難（重新驗證了 main 的 S4 結論：batch 比 event-driven 多做 ~50-100×、SIMD 救不回）。`src/AprVisual/Sim/WireCore.Oblivious.cs`。

### #1 —— Observability merge-pruning（`--prune-merge`）→ **⭐ 1.32×（唯一的突破）**

**唯一打中真瓶頸（D）的、唯一贏的。** 想法：`SetNodeState` 在 gate 變 **HIGH（transistor 開啟 → 兩端 c1,c2 合併成一群）且兩端已同值**時、跳過 re-eval enqueue —— 同值合併是 value-preserving 的（解析優先序在「兩子群都已解析成同值」時不可能翻轉）。**關鍵不對稱：gate 變 LOW（分裂）絕不剪** —— 分裂那刻兩端必然同值、但斷線後可發散（dynamic node 的值是靠連線借來的）。

```csharp
// WireCore.Recalc.cs — SetNodeState, the EnablePruneMerge branch (gate went HIGH)
if (EnablePruneMerge && newState != 0)
{
    while (*p != 0) {
        int c1 = *p++; int c2 = *p++;
        if (NodeStates[c1] != NodeStates[c2]) EnqueueNode(c1);  // skip the equal-value (no-op) merge
    }
}
else  // unchanged S1: ON enqueues c1 (group walk handles c2); OFF (split) enqueues both
{
    while (*p != 0) {
        int c1 = *p++; int c2 = *p++;
        EnqueueNode(c1);
        if (newState == 0 && c2 != Npwr && c2 != Ngnd) EnqueueNode(c2);
    }
}
```

**實測**：D（RecalcNode/hc）`610 → 444`（−27%）→ **1.32×**（bench-hc 50K 和 4-幀穩態都是）。

**等價性（observably-identical、非 bit-identical）**：`--selftest` PASS；CPU trace 1500 cycle 只有 cyc 1-7 不同（power-on `P` 旗標 I-bit，真矽晶片上本來就 indeterminate）、cyc 8+ byte-identical；PPU 渲染輸出（5 幀 screenshot）**MD5 完全相同**；只有 NodeStates 內部的非-observable settle transient 不同（同 main 的 float-artifact 豁免 —— S1 半週期內 settle 在 dynamic/indeterminate 區*不是唯一 fixpoint*）。

## 7. 效能總表 + 結論

| 變體 | flag | hc/s | × baseline | 等價性 |
|---|---|---|---|---|
| baseline S1 | — | ~41-47K | 1.00× | golden |
| G (RCM 重排) | `--rcm` | ~43K | 1.04× | observable identical |
| Y (MLP unroll) | `--simd-queue` | ~41K | 1.00× | observable identical |
| X (Oblivious) | `--oblivious` | 345 | 0.008× | — |
| **#1 (merge-pruning)** ⭐ | `--prune-merge` | **~54K** | **1.32×** | observably-identical（settle transient 豁免）|

**核心結論 —— 瓶頸是 D（dirty-set 大小）、不是 cache、不是 SIMD width**：
- cache 重排（G）≈ 沒用：NodeStates 已塞 L1d。
- SIMD micro-opt（Y）≈ 沒用：byte* 不規則 walk、且不打 D。
- 丟 dirty-set（X）= 121× 倒退：證明 O(D) 是 essential。
- **剪 value-preserving re-eval（#1）→ D 降 27% → 1.32×：唯一打中 D、唯一贏。**

**所以加速 S1 的正道 = 進一步削減 D**（#1 是第一刀；要再往上得繼續找冗餘的 enqueue/recalc、每步驗 observably-identical）。**real-time（~840×）仍不可達** —— 那是 paradigm 的事（IR/codegen，main 的 S2→S4 探索過、結論不可達，見 `MD/RETROSPECTIVE.md`）。但「single-instance switch-level on CPU」的天花板被 #1 從 ~47K 抬到 ~62K hc/s（1.32×），證明 D-reduction 這個 lever 是真的。

—— 也給 Gemini 的預測補一筆：它預測 G（cache）「~1.2-1.5×」、整體「~2×」—— **量級對了（1.3× 真出現了）、但歸錯技術**（gain 來自 D-reduction 不是 cache）。「AI 給 hypothesis、profiler 給 truth」：不只「會不會快」要實測、「為什麼快」也要實測。
