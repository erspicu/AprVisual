你的直覺與觀察非常敏銳！你提到的這點：**「手動 inline 後的 IL code 變小，理論上應該更快，但實測卻變慢」**，正是現代高階編譯器與 JIT（即時編譯器）在面對「超大型方法（Huge Method）」時最經典的效能陷阱。  
你原來的 ProcessQueue 實作雖然是一項非常了不起的工程硬實力展示（試圖在單一方法內融合三種調度路徑與統一的寫回機制），但在 .NET 10/11 的 RyuJIT 視角裡，這種「大一統」的 goto 結構和龐大的區域變數生命週期，反而會破壞 JIT 最核心的最佳化機制。  
我們來進行更深入的底層優化診斷，並解析如何透過調整結構，突破自動 inline 的效能天花板。

## **為什麼手動融合所有路徑反而慢了？**

JIT（RyuJIT）在編譯 ProcessQueue() 時，會面臨以下三個致命的底層物理制約：

### **1\. 暫存器壓力（Register Pressure）暴增與堆疊溢出（Spill）**

在你的手動融合版本中，curList、nextList、curHash、nextHash、nextCount、nodeStates、rS、rA、rB、wbBuf、wbCount 等變數**同時在同一個巨型方法中保持活躍（Live Range 重疊）**。  
CPU 的一般用途暫存器（Registers）數量是有限的（x64 架構下只有 16 個）。當活躍變數超過暫存器數量時，JIT 必須生成大量的 mov 指令，將變數在暫存器和 CPU 堆疊（記憶體）之間來回搬移。這種現象稱為 **Register Spilling**。  
**效能衝擊：** 原本最熱的路徑（如 cls \== 1 的單一節點解析）本該完全在暫存器內完成，現在卻因為巨型方法帶來的暫存器壓力，被迫塞入了記憶體讀寫（Stack access），這會直接吞噬你辛苦優化出來的效能。

### **2\. 共用 WRITEBACK 破壞了路徑特化（Path Specialization）**

為了減少 IL code 大小，你讓 Singleton、B1-pair、BFS 三種截然不同的邏輯在計算完 newState 後，全部跳到同一個 WRITEBACK 標籤。

* **對 Singleton 而言：** 它明明只有 1 個節點，wbCount 永遠是 1。但因為共用了 WRITEBACK，它被迫去執行 for (int m \= 0; m \< wbCount; m++) 的迴圈。雖然迴圈只跑一次，但迴圈的初始化、條件判斷、以及計數器自增，對一個每秒要跑幾十萬次的最熱路徑來說，是非常沉重的開銷。  
* **自動 Inline 的優勢：** 如果維持獨立小方法（如原本分開的 SetNodeState），當它被 inline 到 cls \== 1 的上下文時，JIT 能夠一眼看出「此處的節點數永遠是 1」，進而**自動消除整個 for 迴圈**，將程式碼特化（Specialize）為單一節點的直接寫入。你手動用 goto 融合，反而剝奪了 JIT 進行這種「死碼消除（Dead-code elimination）」與「路徑特化」的機會。

### **3\. 巨型 while-for 內含複雜 goto 導致 JIT 最佳化放棄（Bailout）**

.NET 的 JIT 編譯器在對方法進行控制流分析（Control Flow Analysis）時，如果發現某個方法的基本區塊（Basic Blocks）過多、控制流（goto 與分支）過於錯綜複雜，為了避免編譯耗時過長（JIT 必須在執行期快速完成編譯），它會主動**放棄（Bailout）某些進階最佳化**。例如：

* 迴圈展開（Loop Unrolling）。  
* 更積極的常數傳播（Constant Propagation）。  
* 針對特定硬體指令的優化（例如利用 AVX 進行局部優化）。  
  這就是為什麼你的 IL 變小了，但編譯出來的機器碼（Machine Code）品質反而下降的原因。

## **挑戰編譯器極限：冷熱隔離與最熱路徑特化**

要擊敗（或至少追平）編譯器的自動 inline，重構的策略不應該是「消除所有方法呼叫」，而是「讓最熱的路徑（Fast Path）擁有最純淨、最緊湊、暫存器壓力最小的機器碼，並將冷路徑（Cold Path）徹底隔離」。  
在你的模擬器中，cls \== 1（純邏輯單一節點）與 cls \== 2（且剛好所有閘極都關閉的動態單一節點）佔了絕對多數的事件流。這才是我們需要不計代價保護的「最熱迴圈」。而當它真的要掉進 B1-Pair 甚至是龐大的 BFS 走訪時，這對整個模擬步進來說已經屬於「慢速路徑」了。  
以下是針對你上傳的 WireCore.Recalc.cs 進行演算法與結構調整的優化方案。我們將主迴圈「瘦身」，只保留單一節點的特化寫回，並將 B1-pair 與 BFS 的複雜狀態轉移移至**明確標記為 \[MethodImpl(MethodImplOptions.NoInlining)\] 的冷方法中**：

C\#  
        // ── 優化後的極速排程核心 ──  
        \[MethodImpl(MethodImplOptions.AggressiveOptimization)\] // 提示 JIT 啟動最強編譯  
        private static void ProcessQueue()  
        {  
\#**if** DEBUG  
            int iteration \= 0;  
\#**endif**  
            while (RecalcListNextCount \!= 0)  
            {  
\#**if** DEBUG  
                if (++iteration \== 64)  
                    Console.Error.WriteLine($"WireCore.ProcessQueue: settle pass {iteration} (still propagating, past p99 — see MD/struct/01 §11.2)");  
                if (iteration \> MaxSettlePasses)  
                {  
                    Console.Error.WriteLine($"WireCore.ProcessQueue: aborting after {MaxSettlePasses} settle passes ({RecalcListNextCount} nodes still pending) — leaving state as-is");  
                    for (int i \= 0; i \< RecalcListNextCount; i++) RecalcHashNext\[RecalcListNext\[i\]\] \= 0;  
                    RecalcListNextCount \= 0;  
                    break;  
                }  
\#**endif**  
                // 指標交換與全域變數讀取（維持在波次層級，最大化暫存器利用）  
                int\* curList \= RecalcList; RecalcList \= RecalcListNext; RecalcListNext \= tmpList;  
                byte\* curHash \= RecalcHash; RecalcHash \= RecalcHashNext; RecalcHashNext \= tmpHash;  
                int curCount \= RecalcListNextCount;  
                RecalcListCount \= curCount;  
                RecalcListNextCount \= 0;

                // 將迴圈最常存取的唯讀指標與邊界完全鎖定為區域變數  
                int\* nextList \= RecalcListNext;  
                byte\* nextHash \= RecalcHashNext;  
                int nextCount \= 0; // 這個計數器現在可以完美鎖在單一 CPU 暫存器中（如 R8 或 R9）  
                byte\* nodeStates \= NodeStates;  
                int rS \= RangePruneS;  
                int rA \= RangePruneA;  
                int rB \= RangePruneB;

                // 最熱的核心掃描迴圈  
                for (int i \= 0; i \< curCount; i++)  
                {  
                    int nn \= curList\[i\];  
                    if (curHash\[nn\] \== 0) continue;  
                    curHash\[nn\] \= 0; // 提前清除，騰出邏輯空間

                    byte cls \= IsPureLogic\[nn\];

                    // 🚀 【特化路徑 1】：最常見的純邏輯單一節點 (cls \== 1\)  
                    if (cls \== 1)  
                    {  
                        NodeInfo\* ns \= NodeInfos \+ nn;  
                        int flags \= (int)ns-\>Flags;  
                        if (ns-\>Inline \!= 0)  
                        {  
                            ushort\* pay \= ns-\>InlinePayload;  
                            int gndStart \= ns-\>C1c2Count \<\< 1;  
                            int gndEnd \= gndStart \+ ns-\>GndCount;  
                            int anyG \= 0;  
                            for (int k \= gndStart; k \< gndEnd; k++) anyG |= nodeStates\[pay\[k\]\];  
                            flags |= anyG \<\< 5;  
                            int pwrEnd \= gndEnd \+ ns-\>PwrCount;  
                            int anyP \= 0;  
                            for (int k \= gndEnd; k \< pwrEnd; k++) anyP |= nodeStates\[pay\[k\]\];  
                            flags |= anyP \<\< 4;  
                        }  
                        else  
                        {  
                            if (ns-\>TlistC1gnd \!= 0) { ushort\* p \= TransistorList \+ ns-\>TlistC1gnd; int any \= 0; while (\*p \!= 0) any |= nodeStates\[\*p++\]; flags |= any \<\< 5; }  
                            if (ns-\>TlistC1pwr \!= 0) { ushort\* p \= TransistorList \+ ns-\>TlistC1pwr; int any \= 0; while (\*p \!= 0) any |= nodeStates\[\*p++\]; flags |= any \<\< 4; }  
                        }

                        if (flags \== 0) continue; // 浮接保持原樣，直接跳過寫回

                        byte newState \= FlagsToState\[flags\];  
                        if (nodeStates\[nn\] \!= newState)  
                        {  
                            // 呼叫專為單一節點特化、100% 展開、無內部迴圈的極速寫回  
                            nextCount \= WritebackSingleNode(nn, newState, nextList, nextHash, nextCount, nodeStates, rS, rA, rB);  
                        }  
\#**if** DEBUG  
                        WasteProfileTally(nn, nodeStates\[nn\] \== newState);  
                        CoActivityTally(nn);  
                        BoundaryPopTally(nn);  
\#**endif**  
                        continue;  
                    }

                    // 🚀 【特化路徑 2】：動態單一節點 (cls \== 2\) 的極速快篩  
                    if (cls \== 2)  
                    {  
                        NodeInfo\* ns \= NodeInfos \+ nn;  
                        if (ns-\>Inline \!= 0)  
                        {  
                            ushort\* pay \= ns-\>InlinePayload;  
                            int n2 \= ns-\>C1c2Count \<\< 1;  
                            // 快速檢查是否有任何閘極開啟  
                            for (int k \= 0; k \< n2; k \+= 2)  
                            {  
                                if (nodeStates\[pay\[k\]\] \!= 0)  
                                {  
                                    // 發現開啟的閘極！這代表這不是單一節點，交給獨立的冷方法去處理 B1-Pair 或 Fallback 到 BFS  
                                    nextCount \= ProcessComplexCls2Path(nn, k, n2, ns, curHash, nodeStates, nextList, nextHash, nextCount, rS, rA, rB);  
                                    goto NEXT\_NODE;  
                                }  
                            }  
                        }  
                        else  
                        {  
                            ushort\* p \= TransistorList \+ ns-\>TlistC1c2s;  
                            while (\*p \!= 0)  
                            {  
                                if (nodeStates\[\*p\] \!= 0)  
                                {  
                                    // 高風扇出節點 fallback 到慢速 BFS 處理路徑  
                                    nextCount \= ProcessBfsPath(nn, nodeStates, nextList, nextHash, nextCount, rS, rA, rB);  
                                    goto NEXT\_NODE;  
                                }  
                                p \+= 2;  
                            }  
                        }

                        // 通過快篩，證明此時完全沒有導通管道，它就是一個純 Singleton！  
                        int flags \= (int)ns-\>Flags;  
                        if (ns-\>Inline \!= 0)  
                        {  
                            ushort\* pay \= ns-\>InlinePayload;  
                            int gndStart \= ns-\>C1c2Count \<\< 1;  
                            int gndEnd \= gndStart \+ ns-\>GndCount;  
                            int anyG \= 0;  
                            for (int k \= gndStart; k \< gndEnd; k++) anyG |= nodeStates\[pay\[k\]\];  
                            flags |= anyG \<\< 5;  
                            int pwrEnd \= gndEnd \+ ns-\>PwrCount;  
                            int anyP \= 0;  
                            for (int k \= gndEnd; k \< pwrEnd; k++) anyP |= nodeStates\[pay\[k\]\];  
                            flags |= anyP \<\< 4;  
                        }  
                        else  
                        {  
                            if (ns-\>TlistC1gnd \!= 0) { ushort\* p \= TransistorList \+ ns-\>TlistC1gnd; int any \= 0; while (\*p \!= 0) any |= nodeStates\[\*p++\]; flags |= any \<\< 5; }  
                            if (ns-\>TlistC1pwr \!= 0) { ushort\* p \= TransistorList \+ ns-\>TlistC1pwr; int any \= 0; while (\*p \!= 0) any |= nodeStates\[\*p++\]; flags |= any \<\< 4; }  
                        }

                        if (flags \!= 0)  
                        {  
                            byte newState \= FlagsToState\[flags\];  
                            if (nodeStates\[nn\] \!= newState)  
                            {  
                                nextCount \= WritebackSingleNode(nn, newState, nextList, nextHash, nextCount, nodeStates, rS, rA, rB);  
                            }  
                        }  
                        goto NEXT\_NODE;  
                    }

                    // ❄️ 【冷路徑】：真正的大魔王 BFS 走訪 (cls \== 0\)  
                    nextCount \= ProcessBfsPath(nn, nodeStates, nextList, nextHash, nextCount, rS, rA, rB);

                NEXT\_NODE:  
\#**if** DEBUG  
                    // 在慢速路徑或完成後統一計數，避免污染 cls==1 的熱機器碼  
                    CoActivityTally(nn);  
                    BoundaryPopTally(nn);  
\#**endif**  
                    continue;  
                }  
                RecalcListNextCount \= nextCount;  
            }  
\#**if** DEBUG  
            SettlePassTally(iteration);  
\#**endif**  
            InvokeCallbacks();  
        }

        // ── 專為單一節點特化的極速寫回（無任何 wbBuf 迴圈開銷，徹底解除暫存器壓力） ──  
        \[MethodImpl(MethodImplOptions.AggressiveInlining)\]  
        private static unsafe int WritebackSingleNode(int nn, byte newState, int\* nextList, byte\* nextHash, int nextCount, byte\* nodeStates, int rS, int rA, int rB)  
        {  
            nodeStates\[nn\] \= newState;  
\#**if** DEBUG  
            DiagStateChanges++;  
            if (CutNodeKind \!= null) { int \_ck \= CutNodeKind\[nn\]; if (\_ck \!= 0) DiagCut\[\_ck\]++; }  
\#**endif**  
            int tg \= NodeTlistGates\[nn\];  
            if (tg \!= 0)  
            {  
                ushort\* p \= TransistorList \+ tg;  
                if (newState \== 0)  
                {  
                    while (true)  
                    {  
                        ulong quad \= Unsafe.ReadUnaligned\<ulong\>(p);  
                        int c1a \= (ushort)quad; if (c1a \== 0) break;  
                        int c2a \= (ushort)(quad \>\> 16);  
                        if (c1a \>= rS && nextHash\[c1a\] \== 0) { nextList\[nextCount++\] \= c1a; nextHash\[c1a\] \= 1; }  
                        if (c2a \>= rS && nextHash\[c2a\] \== 0) { nextList\[nextCount++\] \= c2a; nextHash\[c2a\] \= 1; }  
                        int c1b \= (ushort)(quad \>\> 32); if (c1b \== 0) break;  
                        int c2b \= (ushort)(quad \>\> 48);  
                        if (c1b \>= rS && nextHash\[c1b\] \== 0) { nextList\[nextCount++\] \= c1b; nextHash\[c1b\] \= 1; }  
                        if (c2b \>= rS && nextHash\[c2b\] \== 0) { nextList\[nextCount++\] \= c2b; nextHash\[c2b\] \= 1; }  
                        p \+= 4;  
                    }  
                }  
                else  
                {  
                    while (true)  
                    {  
                        ulong quad \= Unsafe.ReadUnaligned\<ulong\>(p);  
                        int c1a \= (ushort)quad; if (c1a \== 0) break;  
                        int c2a \= (ushort)(quad \>\> 16);  
                        if ((c1a \< rA || c1a \>= rB || nodeStates\[c1a\] \!= nodeStates\[c2a\]) && nextHash\[c1a\] \== 0) { nextList\[nextCount++\] \= c1a; nextHash\[c1a\] \= 1; }  
                        int c1b \= (ushort)(quad \>\> 32); if (c1b \== 0) break;  
                        int c2b \= (ushort)(quad \>\> 48);  
                        if ((c1b \< rA || c1b \>= rB || nodeStates\[c1b\] \!= nodeStates\[c2b\]) && nextHash\[c1b\] \== 0) { nextList\[nextCount++\] \= c1b; nextHash\[c1b\] \= 1; }  
                        p \+= 4;  
                    }  
                }  
            }  
            return nextCount;  
        }

        // ── 【冷路徑隔離 1】：處理 cls \== 2 且伴隨有電晶體導通的複雜情況（B1-Pair 或 Fallback） ──  
        \[MethodImpl(MethodImplOptions.NoInlining)\]  
        private static unsafe int ProcessComplexCls2Path(int nn, int firstOnIdx, int n2, NodeInfo\* ns, byte\* curHash, byte\* nodeStates, int\* nextList, byte\* nextHash, int nextCount, int rS, int rA, int rB)  
        {  
            ushort\* pay \= ns-\>InlinePayload;  
            int o \= pay\[firstOnIdx \+ 1\];  
              
            // 實作原本你寫在 ProcessQueue 內的 B1 Pair 驗證  
            if (o \== nn) goto FallbackBFS;  
            for (int k2 \= firstOnIdx \+ 2; k2 \< n2; k2 \+= 2) if (nodeStates\[pay\[k2\]\] \!= 0) goto FallbackBFS;  
              
            NodeInfo\* os \= NodeInfos \+ o;  
            if (os-\>Inline \== 0 || (os-\>Flags & (NodeFlags.HasCallback | NodeFlags.ForceCompute)) \!= 0) goto FallbackBFS;  
              
            ushort\* opay \= os-\>InlinePayload;  
            int on2 \= os-\>C1c2Count \<\< 1;  
            for (int k2 \= 0; k2 \< on2; k2 \+= 2) if (nodeStates\[opay\[k2\]\] \!= 0 && opay\[k2 \+ 1\] \!= nn) goto FallbackBFS;

            // 成功認證！這是一個精確的 B1 Pair 組  
            curHash\[o\] \= 0;  
            int flags \= (int)ns-\>Flags | (int)os-\>Flags;  
            int anyG \= 0, anyP \= 0;  
            int sGe \= n2 \+ ns-\>GndCount, sPe \= sGe \+ ns-\>PwrCount;  
            for (int j \= n2; j \< sGe; j++) anyG |= nodeStates\[pay\[j\]\];  
            for (int j \= sGe; j \< sPe; j++) anyP |= nodeStates\[pay\[j\]\];  
            int oGe \= on2 \+ os-\>GndCount, oPe \= oGe \+ os-\>PwrCount;  
            for (int j \= on2; j \< oGe; j++) anyG |= nodeStates\[opay\[j\]\];  
            for (int j \= oGe; j \< oPe; j++) anyP |= nodeStates\[opay\[j\]\];  
            flags |= (anyG \<\< 5) | (anyP \<\< 4);  
              
            byte v \= flags \!= 0 ? FlagsToState\[flags\]  
                   : (NodeConnections\[o\] \> NodeConnections\[nn\] ? nodeStates\[o\] : nodeStates\[nn\]);

            // 分別呼叫兩次特化的單一節點寫回，維持 Gauss-Seidel 順序  
            if (nodeStates\[nn\] \!= v) nextCount \= WritebackSingleNode(nn, v, nextList, nextHash, nextCount, nodeStates, rS, rA, rB);  
            if (nodeStates\[o\] \!= v) nextCount \= WritebackSingleNode(o, v, nextList, nextHash, nextCount, nodeStates, rS, rA, rB);

\#**if** DEBUG  
            DiagPairPath++;  
            WasteProfileTally(nn, false); // 簡化處理  
\#**endif**  
            return nextCount;

        FallbackBFS:  
            return ProcessBfsPath(nn, nodeStates, nextList, nextHash, nextCount, rS, rA, rB);  
        }

        // ── 【冷路徑隔離 2】：標準大魔王隨機圖 BFS 走訪與寫回（強制不內聯，釋放主迴圈暫存器） ──  
        \[MethodImpl(MethodImplOptions.NoInlining)\]  
        private static unsafe int ProcessBfsPath(int nn, byte\* nodeStates, int\* nextList, byte\* nextHash, int nextCount, int rS, int rA, int rB)  
        {  
            byte newState \= ComputeNodeGroup(nn);  
            int gc \= \_groupCount;  
            ushort\* gb \= \_groupBuf;  
              
            // 這裡保留你原本優化過的、針對群組陣列的優化寫回迴圈  
            if (newState \== 0)  
            {  
                for (int m \= 0; m \< gc; m++)  
                {  
                    int gn \= gb\[m\];  
                    if (nodeStates\[gn\] \== 0) continue;  
                    nodeStates\[gn\] \= 0;  
\#**if** DEBUG  
                    DiagStateChanges++;  
                    if (CutNodeKind \!= null) { int \_ck \= CutNodeKind\[gn\]; if (\_ck \!= 0) DiagCut\[\_ck\]++; }  
\#**endif**  
                    int tg \= NodeTlistGates\[gn\];  
                    if (tg \== 0) continue;  
                    ushort\* p \= TransistorList \+ tg;  
                    while (true)  
                    {  
                        ulong quad \= Unsafe.ReadUnaligned\<ulong\>(p);  
                        int c1a \= (ushort)quad; if (c1a \== 0) break;  
                        int c2a \= (ushort)(quad \>\> 16);  
                        if (c1a \>= rS && nextHash\[c1a\] \== 0) { nextList\[nextCount++\] \= c1a; nextHash\[c1a\] \= 1; }  
                        if (c2a \>= rS && nextHash\[c2a\] \== 0) { nextList\[nextCount++\] \= c2a; nextHash\[c2a\] \= 1; }  
                        int c1b \= (ushort)(quad \>\> 32); if (c1b \== 0) break;  
                        int c2b \= (ushort)(quad \>\> 48);  
                        if (c1b \>= rS && nextHash\[c1b\] \== 0) { nextList\[nextCount++\] \= c1b; nextHash\[c1b\] \= 1; }  
                        if (c2b \>= rS && nextHash\[c2b\] \== 0) { nextList\[nextCount++\] \= c2b; nextHash\[c2b\] \= 1; }  
                        p \+= 4;  
                    }  
                }  
            }  
            else  
            {  
                for (int m \= 0; m \< gc; m++)  
                {  
                    int gn \= gb\[m\];  
                    if (nodeStates\[gn\] \!= 0) continue;  
                    nodeStates\[gn\] \= 1;  
\#**if** DEBUG  
                    DiagStateChanges++;  
                    if (CutNodeKind \!= null) { int \_ck \= CutNodeKind\[gn\]; if (\_ck \!= 0) DiagCut\[\_ck\]++; }  
\#**endif**  
                    int tg \= NodeTlistGates\[gn\];  
                    if (tg \== 0) continue;  
                    ushort\* p \= TransistorList \+ tg;  
                    while (true)  
                    {  
                        ulong quad \= Unsafe.ReadUnaligned\<ulong\>(p);  
                        int c1a \= (ushort)quad; if (c1a \== 0) break;  
                        int c2a \= (ushort)(quad \>\> 16);  
                        if ((c1a \< rA || c1a \>= rB || nodeStates\[c1a\] \!= nodeStates\[c2a\]) && nextHash\[c1a\] \== 0) { nextList\[nextCount++\] \= c1a; nextHash\[c1a\] \= 1; }  
                        int c1b \= (ushort)(quad \>\> 32); if (c1b \== 0) break;  
                        int c2b \= (ushort)(quad \>\> 48);

## **優化後的本質改變（為何這能擊敗原版大一統方法）**

1. **核心熱迴圈機器碼精簡至極致：** ProcessQueue 內部的 for 迴圈現在變得異常乾淨。針對最普遍的 cls \== 1，它的邏輯線路是直接且無分支交叉的。JIT 能夠將全部的焦點放在這個小迴圈上。  
2. **暫存器完美鎖定：** 由於把 BFS 迴圈和複雜判斷抽離，主迴圈內的變數數量大減。JIT 終於有能力把關鍵的 nextCount 放入 CPU 暂存器中，在整個波次的 for 執行期間，它不需要發生任何堆疊溢出寫回。  
3. **消除無意義的跌代：** WritebackSingleNode 是完全針對「已知節點數為 1」展開的。它沒有 for (int m \= 0; m \< wbCount) 的間接開銷，直接對電晶體網表進行指標追蹤，抹平了原版寫回時的框架開銷。  
4. **冷熱隔離（PGO 友善）：** 當編譯器看到 ProcessComplexCls2Path 與 ProcessBfsPath 被標記為 NoInlining 時，它會把這兩段龐大的機器碼移到主程式碼段（Hot Section）以外的冷區域（Cold Section）。這極大地提高了 CPU 的 **指令快取（i-cache）命中率**，讓最熱的單一節點電路能以最順暢的流水線（Pipeline）在 Zen 2 上奔跑。

這套結構本質上是在**利用現代編譯器的強項（冷熱分離與區域暫存器分配），同時手工消滅其弱點（虛擬調度與多路徑寫回開銷）**。你可以嘗試將這個重構後的結構放入你的 AprVisual 進行消融實驗，觀察硬體計數器中的暫存器溢出與執行時鐘週期，理論上它具備了與 JIT 自動 inline 甚至超越其極限的打擊力！
