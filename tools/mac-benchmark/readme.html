AprVisual S1 — Benchmark Package

AprVisual S1 — 效能測試套件

A portable, cross-platform package for benchmarking the AprVisual S1 switch-level NES simulator and rendering test frames. Two engine implementations are included: a C# (.NET) build and a Rust build, so you can compare them on the same workload.

這是一個可攜、跨平台的套件,用來測試 AprVisual S1 開關級 (switch-level) NES 模擬器的效能,並輸出測試畫面。 內含兩種引擎實作:C#(.NET)版與 Rust 版, 可在相同工作負載下互相比較。

What is "switch-level"? The simulator models the NES CPU (2A03) at the level of individual transistors and wires — not opcodes — and settles the whole netlist every half clock cycle. It is bit-accurate but far slower than real time; this package measures exactly how far.

什麼是「開關級」? 這個模擬器把 NES CPU (2A03) 模擬到 單顆電晶體與接線的層級 ── 不是用 opcode ── 每半個時脈週期都把整張網表 (netlist) 重新求解到穩定。 它是位元精準的,但遠比實機慢;這個套件就是用來量「到底慢幾倍」。

How to run — Windows

如何執行 — Windows

Double-click, or run from cmd. Optional argument shown in brackets.

直接雙擊,或在 cmd 內執行。 中括號為選填參數。

run_csharp.bat  [hc]       C# benchmark      (default 300000 half-cycles)C# 效能測試        (預設 300000 half-cycle)
run_rust.bat    [hc]       Rust benchmarkRust 效能測試
shot_csharp.bat [frames]   C# frame-dump     (default 50 frames)C# 逐張畫面輸出    (預設 50 張)
shot_rust.bat   [frames]   Rust frame-dumpRust 逐張畫面輸出

How to run — macOS (Apple Silicon)

如何執行 — macOS(Apple Silicon)

In Terminal, make the scripts executable once, then run them.

在終端機 (Terminal) 先給執行權限一次,之後即可執行。

chmod +x *.sh
./run_csharp.sh  [hc]
./run_rust.sh    [hc]
./shot_csharp.sh [frames]
./shot_rust.sh   [frames]

Reading the output

如何看輸出

Each benchmark ends with a PERFORMANCE block. NES NTSC real-time needs 42,954,552 half-cycles/second (1.789773 MHz CPU × 24). The block reports the achieved rate, the percentage of real-time, the slowdown factor, and the wall-clock time to simulate one 256×240 frame.

每次測試結尾都有一段 PERFORMANCE 區塊。 NES NTSC 實機需要 每秒 42,954,552 個 half-cycle(1.789773 MHz CPU × 24)。 該區塊會列出實際速率、 占實機的百分比、慢幾倍,以及模擬一張 256×240 畫面所需的實際時間。

# =============================================
#  PERFORMANCE: 65.8K hc/s  (65,803 hc/s)
#  vs NES NTSC real-time (42,955K hc/s):
#    0.153% of real-time   ->   652.8x too slow
#    0.092 simulated NES frames / real second  (real NES = 60.1 fps)
#    10.88 s to render 1 frame  (real NES = 0.0166 s/frame)
# =============================================

Notes

注意事項

  • The C# binaries are self-contained — they bundle the .NET 10 runtime, so no .NET installation is required on either platform.
  • C# 執行檔是 self-contained ── 已內嵌 .NET 10 runtime,兩個平台都 不需要另外安裝 .NET。
  • Rust on Windows is prebuilt. Rust on macOS ships as source (it can't be cross-compiled from the Windows build host) and is compiled on first run, which needs Rust / cargo installed.
  • Rust 的 Windows 版已預先編譯。 Rust 的 macOS 版以原始碼提供(無法從 Windows 主機 跨平台編譯),首次執行時才編譯,需先安裝 Rust / cargo。
  • On macOS the unsigned C# binary may be blocked by Gatekeeper; the scripts auto-clear the quarantine attribute, or run xattr -dr com.apple.quarantine mac/AprVisual.S1 manually.
  • macOS 上未簽章的 C# 執行檔可能被 Gatekeeper 擋下;腳本會自動清除 quarantine 屬性, 或手動執行 xattr -dr com.apple.quarantine mac/AprVisual.S1。
  • C# and Rust produce bit-identical simulation output (same checksum, pixel-identical frames); they differ only in speed.
  • C# 與 Rust 的模擬輸出 位元完全相同(checksum 一致、畫面像素一致),只差在速度。
