## 0. 動機與定位

**S1 的現況**:二值(0/1)、settle-to-quiescence 的開關級引擎,對「參考模擬器」bit-exact、最快
(~80–100K hc/s)。但為了對**真實矽晶片** bit-exact(AccuracyCoin 141/141),累積了 **13 個 shim**
+ 1 個網表補丁(E 類)。每個 shim 補一種「二值抽象表達不出的類比行為」,而且多半是 per-test 觸發、
針對性強。這正是這份文件要處理的:**shim 是症狀,類比才是病**。

**S1a 的定位**(對照 CLAUDE.md 的專案 thesis):
- 專案結論之一:**~98.9% 活動可化約為 logic+register,~1.1% 是真正的類比**。13 個 shim 就是在補那 1.1%。
- 全類比模擬(解微積分)= 太慢、不可行(見 `WebSite/semantic-ceiling.html`)。**S1a 不推翻這個 thesis。**
- S1a 的問題是:**那 1.1% 的類比,用「最小必要的類比機制」去描述,需要幾個機制、多少額外成本?**
  這是一個可量化的研究問題。S1a 會比 S1 慢(類比島有成本),這是**刻意的 accuracy/principled trade**,
  不是退步。S1 保持 golden 速度不動。

**一句話**:S1a = 把兩戰役累積的 ~20 個「你怎麼知道要這樣 patch」的黑箱 shim,換成 7 個「這是它物理上
怎麼運作」的白箱機制(M1–M7)。通則化(使用者要的)= 機制取代 shim。**目標函數(承 2026-07-14 通解
架構):不是「零 shim」(A2/A4 連續時間本質不可達),而是「零手寫特例」—— 沒有一行認站點/認測試的代碼。**

## 2. 設計原則:混合訊號(digital core + analog islands)

1. **保留快速二值核心跑 98.9%**。不動 group BFS + GND-wins LUT 的 fast path。
2. **類比集中在「類比島」(analog islands)**:載入期標記出 binary 抽象會失效的少數子電路
   (比例戰節點群、無 pull-up 的電荷儲存節點、跨晶片延遲網、透明閂鎖、板級元件)。
3. **島求解器只在島節點活躍時觸發**(bounded)。fast path 不背成本;島的成本只在該島有事時付。
4. **誠實面對 thesis**:全類比太慢。S1a 求的是「島數量最少、每島最便宜」的最小擴充,不是把整顆晶片變 SPICE。

**判定「是不是島」的載入期分析(自動化目標)**:
- 節點群同時有 pull-up 與 pull-down、且無明確 GND 勝出 → **M1 比例戰島**候選。
- 節點無 pull-up、gate-list 有觀察者、靠電容保持 → **M2 電荷島**候選(OAM cell、匯流排)。
- 跨晶片高扇出網(CPU→PPU 控制、AD 匯流排) → **M3 延遲島**候選。
- pass-gate + 回授到自身 gate/上游 → **M4 透明閂鎖島**候選。
- 板級外部連線(卡帶/主機板 TTL) → **M5 板級元件**。

## 3. 七個機制(M1–M7:M1–M5 類比、M6 相位、M7 決定論)

### M1 — 電導加權群解析(drive-strength resolution)
**取代**:LxaMagicShim、AluLatchShim(A1 部分)。
**現況**:group 解析用 flags-OR → 256-entry LUT,GND 一律勝。這對「比例戰」(兩邊都導通、靠 W/L 比
決勝)是錯的 —— 所以 LXA 的 magic 常數要用 shim 硬灌 $FF。
**機制**:給每個電晶體一個電導權重(來自 transdefs 幾何 geom[5] 的 W/L,或第 7 欄 weak/depletion flag
—— 目前 `IsWeak` 解析了卻沒用)。group 解析改成**比較 pull-up 總電導 vs pull-down 總電導**(分壓),
弱側輸、比例接近時進入類比中間態(用門檻二值化,或標記為需 M2 電荷解析)。
**成本**:只在「雙邊導通」的 group 付(census 顯示這種 group 很少)。fast path 的單邊 group 不變。
**開放問題**:W/L 幾何資料的可靠度;magic=$FF 是否真能從電導自然浮現(需 census 驗證候選)。

### M2 — 節點電容 + 電荷儲存/衰減(charge storage)
**取代**:OpenBusShim、OamBlankEdgeShim。
**現況**:floating group 目前「最大電容節點勝」是靜態的,沒有「電荷會被讀取刷新 / 會衰減」的動態。
所以 open bus(最後位元組)和 OAM 動態 cell(讀取即刷新、關渲染邊沿毀損)要用 shim。
**機制**:每個節點掛一個電容值(**`tools/netlist/node_area.py` 已能從 segdefs 算分層多邊形面積當電容代理**)。
floating 節點保持「最後被驅動值」,並可選擇性衰減(leakage);被讀取路徑(bit-line precharge + sense)
時刷新。open bus = 匯流排電容節點的保持值;OAM cell = 無 pull-up 的交叉耦合對,靠讀取刷新續命。
**成本**:只在 floating / 動態儲存節點付。多數節點有明確驅動,不進此路徑。
**開放問題**:衰減時間常數的校準(用 hc 為單位,先粗後細);哪些節點算「動態儲存」需載入期標記。

### M3 — RC 傳播延遲(per-net timing)【最難、最關鍵】
**取代**:PpuWriteDelay、PpuWriteDelayGlobal(dot-339);**是 ALERead / CPU-PPU 同步相位的深層根因**。
**現況**:settle-to-quiescence 在**一個 half-cycle 內**把狀態瞬間傳到穩定,**沒有 net 內傳播延遲概念**。
跨晶片延遲(dot-339 早 3 dot、ALERead 早 1 CPU cycle)因此被抹零。dot-339 shim 是粗暴版:把單一訊號
固定延遲 24hc。
**機制**:給**特定跨晶片/高扇出網**一個 RC 延遲模型(delay ≈ 驅動電導 × 下游電容,非對稱:
NMOS 上拉慢/下拉快 → 解釋 dot-339 的 16hc rise / 18hc fall)。node 轉態時,對這些網的下游轉態
排程在 t + delay,而非同一 settle 內。**關鍵:只對「載入期標記的少數延遲島」做事件排程,其餘維持
instant-settle** —— 否則就是整顆晶片 timing-accurate,太慢(thesis)。
**成本**:延遲島需要一個「延遲事件佇列」(排程 t+delay 的轉態)。島愈少愈便宜。
**開放問題(重大)**:
- ALERead 的根因是「$2007 讀取的巨觀指令時序早 1 CPU cycle」,Gemini 判斷可能是 **NMI/$2002 輪詢
  同步相位** 或 **開機相位對齊(power-on phase,1 cycle=3 dot,4 種可能相位)**。**這可能不是 net RC
  延遲能修的**,而是 CPU 中斷取樣的 T-state 邊界判定 / 開機相位決定論。→ M3 未必涵蓋 ALERead;
  ALERead 可能需要獨立的「CPU-PPU 相位校準」機制(見 §7 開放問題)。
  **→ 已解答(2026-07-16)**:ALERead **正是**介面延遲問題,M3×M6 事件級存取延遲可修 —— io_ce mux
  單點原型實證(swallow+replay+ale-freeze,24hc,branch aleread-ioce-mux 89370de,孤立 1/1 PASS +
  雙 stress 鄰居全綠)。不需要獨立相位校準機制。實作策略見 **`01_時序標註網表_物理延遲參數化_
  M3M6實作參考.md`**(事件級標註;rise/fall 牆;split/replay opt-in 紀律)。
- 這是 S1a 最大的技術賭注:能不能只延遲少數島、又保持大部分 fast path?dot-339 的成功(單訊號延遲)
  是正面證據;但一般化到「自動標記延遲島 + 校準每島 RC」還沒被證明可行。

### M4 — 亞週期透明閂鎖 + 回授(sub-cycle transparent latch)
**取代**:DlShim、DmcLatchShim、Dmc4015AbortShim、FrameIrqShim、Dbl2007Shim、OamDmaPpuBusShim、
BoardOctalLatch 的回授部分(6+ 個 shim,最大宗)。
**現況**:透明閂鎖(pass-gate,LE 高透明、低保持)+ 回授迴圈,在「一個 settle 一次」的粒度下,
關門沿的賽跑、φ2 透明、正回授鎖定都表達不出來 → 一堆 edge-capture / hold shim。
**機制**:把透明閂鎖建成一級物件(transparent-latch primitive):LE 高時輸出跟隨輸入並捕捉、LE 低時保持;
**在 half-cycle 解析度**下更新(不是一個 settle 一次)。回授迴圈(如 74LS373 的 ALE+/RD)用
**小型 relaxation**(迭代到不動點,已有 `HasNonTrivialRomFeedbackCycle` 的 Floyd 雛形)解析。
**成本**:半週期更新只加在被標記為透明閂鎖的節點上。
**開放問題**:哪些 pass-gate 算「需半週期建模的透明閂鎖」需載入期偵測(gate 連到自身/上游 = 回授指紋);
和 M3(延遲)的交互(閂鎖的賽跑本質是延遲競爭)。

### M5 — 板級元件庫(board-level component library)
**取代**:BoardOctalLatch(74LS373 本體)、OamDmaPpuBusShim(部分)。
**現況**:S1 的節點圖**只含 2A03 + 2C02 晶片內部**,不含卡帶/主機板上的外部 TTL(74LS373 八位閂鎖、
位址解碼)。所以 ALERead 的 74LS373 效應要用 shim 在 CHR 回呼裡模擬。
**機制**:建一個板級元件庫,把外部 TTL 當**一級元件**掛在晶片之間的匯流排上:
`Ls373OctalLatch`(AD0-7 多工解多工)、`Ls139Decoder`(位址解碼)等。每個元件有自己的 pin 語意
(LE/OE/D/Q),在 M4 的半週期粒度下更新。這樣 74LS373 回授是**元件自然行為**,不是 CHR 回呼裡的 hack。
**成本**:板級元件數量少(NROM 板就一顆 '373 + 解碼),成本可忽略。
**開放問題**:元件的 datasheet 級時序(propagation delay、透明 window)要不要精確建(接 M3);
板級網表從哪來(MetalNES 的 `system-def/*.js` 有 board TTL 定義可參考)。

**★ 旗艦案例:ALERead(boing2k7)—— M5 精確破「純開關級天花板」(2026-07-15 六輪 Gemini 收束)**:
ALERead 是整個 141 顆裡**唯一同時踩滿 A4(跨晶片延遲)× B(74LS373 未建模)× 二值架構限制
(GND>VCC>Drive → 無法注入匯流排 '1'、無法延遲被驅動信號)**三層的。純開關級**證明不可修**
(BOL 下游 shim 資料相依過度開火打壞 2007Stress;上游延遲撞 force-high 牆 —— 見
`MD/toDoNext4/202607150004` §五-b/c、六輪諮詢 a_bol_*/a_m6_*)。**但 M5 可精確破**:
- 不只 `Ls373OctalLatch`(行為層 code byte,**繞過 force-high 牆**),還要配 **行為層 $2007 SR pipeline**
  (readLatch/writeLatch + ReadALE)給**正確相位的 ALE** → 八位閂鎖在 ppuAddressBus=$2FFF 那刻捕捉 $FF。
  相位用 offset 校準(AprNes offset=7)。**現成參考碼**:`AprNesRef/AprNes/NesCore/ppu_new.cs`
  (PPU_DATA_Pipeline_Step1)、`ppu_dispatch.cs`(ppuOctalLatch 更新 + `(baseAddr&0xFF00)|ppuOctalLatch`)。
- **為什麼 M5 精確而 BOL/虛擬 373 不精確**:AprNes SR pipeline 給精確 $2007-read 相位(非時間窗啟發式、
  非抓早相位的 ppu.ab)→ 只在 boing2k7 真發生時開火 → 2007Stress 零誤傷。
- **意義**:把「140/141 天花板」精確化成「**純開關級**天花板;M5 合法行為層板級模型可破 → 141/141」。
  這是 M5 最有力的旗艦案例,證明「板級元件庫」不是錦上添花,而是**破純開關級天花板的關鍵**。

### M6 — 上電狀態 + CPU-PPU 相位決定論(power-on & phase)
**取代**:`--reset-hold-extra`(NMI-edge 家族 8 測)、power_up_palette、registers 上電注入;
**是 ALERead CPU-PPU 相位的候選解**。
**現況**:二值模型把兩件類比事實量化掉:(a) **上電節點是隨機的**(真機開機 palette/暫存器/旗標無定義);
(b) **CPU÷12 與 PPU÷4 的相對相位**是離散抽籤(§2.5:只有 4 種 {1,7,5,3},偶數相位物理上不存在)。
目前用一次性注入(power_up_palette 32 cell 雙側)+ 固定對齊(`--reset-hold-extra 1` = 對齊 7,blargg 校準相位)補。
**機制**:把「上電初態 + 相對相位」做成**一級可設定的決定論參數**:
- **上電狀態表**:一張「節點 → 上電值」的資料表(dynamic storage 節點的初始電荷),取代散落的注入 shim。
- **相位選擇器**:把 CPU/PPU 除頻器的相對相位當一個明確參數(4 種),而非隱含於 reset 時序。
  掃 4 種相位、選對釘死機型(NES-001 + G 版)的那個 —— **這正是 blargg 的 golden alignment**。
**成本**:近乎零(載入期設定,不進 hot path)。
**開放問題(重大,連到 ALERead)**:
- §2.5 再定性(Gemini q/a_even_odd)指出:真機有 golden alignment 單次上電十項全過 → 我們的「零交集」
  是**雙 netlist 板級整合的 ~1-dot 絕對相位偏移(未仲裁)**,候選源 = $2002 跨晶片讀取路徑理想化為零延遲 /
  reset 釋放啟動 PPU 除頻器的時序 / 時鐘 pad 緩衝延遲未建模。
- **ALERead**($2007 讀取早 1 CPU cycle)極可能是**同一個未仲裁的跨晶片相位偏移**:若 M6 的相位選擇器 +
  M3 的 $2002/$2007 讀取路徑延遲能把這個 ~1-cycle 偏移仲裁掉,ALERead 的 BoardOctalLatch 觸發窗就能
  對準真 dot(而非現在靠 read_2007_trigger + 窗口的啟發式)。→ **M6×M3 是否能一次解掉 even_odd + dot-339 +
  ALERead 三者的共同相位根因,是 S1a 最有價值的統一假說。**
- 仲裁實驗(知識庫已列):PPUSim 交叉比對 BIT $2002 的絕對 master-clock 延遲(/RD→D7)。

**M2/M4 的共同地基 —— 儲存元件四質地(知識庫 §2.4,施力點選擇的血淚表)**:

| 質地 | 例子 | 單發 force 黏不黏 | S1a 機制 |
|---|---|---|---|
| pass-gate 動態閂鎖 | pcm_latch、alua/alub、A/X 暫存器 | ✅ 門關後浮接保持 | **M4**(半週期透明閂鎖) |
| 交叉耦合再生對 | palette cell、pcm_irq | ❌ 單側被彈回 | **M2**(電荷)+ 雙側解析 |
| 主動再生迴路 | P 旗標(每 cycle 從上游重載) | ❌ 一 cycle 內被沖 | **M4** 迴路 relaxation + 持續驅動 |
| 閘級 TTL 閂鎖模組 | nes-pad CD4021 | ❌ 結構性不可驅動 | **M5**(板級元件)或行為層 |

> 這張表是 M2/M4/M5 「該用哪種機制」的判定依據:**質地決定機制**,不是 per-node 猜。

### M7 — 正準重編號(canonical renumbering)【D 類樂透通則解;原設計漏掉、據 AccuracyEpoch 提案補上】
**取代**:D 類「id-order 樂透」的根源 —— 不是某個 shim,而是**引擎用節點編號(載入順序)裁決同波賽跑**
的結構性缺陷(u7/u8 孿生一好一壞、OpenBus err6、圖變更重擲全域樂透)。
**現況**:二值 settle 波內「同時」事件的處理順序 = 佇列順序 = 編號順序 = 載入/檔案列順序(任意)。
浮接群平手也用編號決勝。→ 真矽用連續時間物理裁決,引擎用**任意編號**裁決。
**機制(載入期一次,熱路徑零改動)**:把節點編號從「載入順序的任意標籤」換成「按物理正準鍵排定的
確定性標籤」——`key = (class, layeredArea, structHash, degree)` 字典序(class 保 locality 分桶不動,
桶內換鍵)。既有 id-順序機制**原封不動地變成正準仲裁**。文獻:Nauty 式正準圖標記 + MOSSIM II 面積代理。
**兩個獨立收益**:
1. **孿生必同命**:同構電路(u7/u8、joy 通道、db 位元)→ 同鍵排序 → 同判決 → 樂透根除。
2. **圖變更不再重擲全域樂透**:編號由結構+物理決定,不依載入順序 → 補一顆管/掛儀器只擾動局部
   → **Socket Pattern 的「載入期圖變更=大爆炸」問題大幅消解**(這個操作面收益可能比 D 類本身更值錢:
   未來 netlist patch、模組置換、儀器掛載全變安全操作)。
**成本**:近乎零(載入期一次排序;可與既有 class-major renumber +3.56% locality win 合成)。
**開放問題**:一致 ≠ 正確(正準化根除「樂透性」,但同命判決是否符合矽取決於鍵與物理的相關性;
面積抓不到路徑電阻/驅動強度的維度);checksum 必變(仲裁換鍵)。
**與 M6 的分工**:M7 = **同一 settle 波內同時事件的空間仲裁**(誰先處理);M6 = **上電初態 + 跨半週期
的相對相位**(從哪個相位起跑)。兩者都是「決定論化非物理的任意性」,但一個管波內順序、一個管相位。

## 5. 驗證策略(bit-exact 為地基)

1. **Phase 0 閘:S1a(類比全關)≡ S1**。金 checksum 必須維持 `0x794A43ABDF169ADA`(full_palette

2. **每個機制上線時**:退休對應 shim,跑該 shim 覆蓋的孤立測試 + 全套 AC 回歸,**不得退步**。
   驗收標準 = **AC 141/141 without per-test shim**(用機制取代,不是關掉)。
3. **參考模擬器 bit-exact**:數位部分仍要對 MetalNES/VisualNes bit-exact(跨模型 checksum)。
4. **交叉驗證**:S1a 的機制輸出應**通則地**重現 shim 的結果(例:M2 的電荷模型應自然產生 open-bus
   的 last-byte,而非硬寫)。

## 7. 風險與開放問題(誠實清單)

1. **效能**:S1a 必然比 S1 慢。要量化「每個機制的島成本」,確保島夠稀疏(fast path 保住)。若某機制
   拖垮整體 → 該行為退回 shim(接受局部黑箱)。**S1 永遠是 golden 速度基準,S1a 不取代它。**
2. **M3 事件排程 vs settle-to-quiescence**:這是架構級賭注。dot-339 單訊號延遲成功是正面證據,但
   自動標記延遲島 + 校準每島 RC 常數還沒證明可行。若不可行 → M3 退回「少數手標延遲網 + 固定延遲」
   (等於把 dot-339 shim 通則成一張延遲網表,仍比 per-test 好)。
3. **ALERead 的真根因**:「$2007 讀取早 1 CPU cycle」可能是 **CPU 中斷/$2002 取樣的 T-state 邊界** 或
   **開機相位決定論**,**未必是 net RC 延遲**。→ 可能需要獨立的「CPU-PPU power-on 相位校準」機制
   (掃 4 種相位、選對 AC 套件的那個),而非 M3。這是 S1a 要回答的一個具體研究問題。
4. **校準資料來源**:M1 的 W/L、M2/M3 的電容/RC 常數,從幾何(segdefs/transdefs)推的是代理值,
   不是真 SPICE 參數。準確度上限受此限制;先求「方向對 + 二值化門檻對」,不求連續電壓精確。
5. **通則會不會真的收斂**:假設「13 shim → 5 機制」可能過度樂觀;實作中可能發現某些行為機制之間耦合
   (如 M3×M4 的閂鎖賽跑),需要更細的子機制。這本身就是有價值的發現。

## 8. 與專案 thesis 的關係(學術定位)

- **不推翻 thesis**:S1(二值)是最快、golden;全類比太慢仍成立。
- **S1a 量化 thesis 的另一半**:「那 1.1% 類比,用最小必要機制描述,是 **7 個機制(M1–M7)**、島佔比多少、
  成本多少」。這把「1.1% 不可約」從一個**比例數字**,變成一份**機制清單 + 成本帳**;而且證明兩戰役
  (nes-test-roms 146/1 + AC 141)累積的 ~20 項行為層處理,**收斂到 7 種物理/決定論現象**,不是散亂的 patch。
- **目標函數(承 2026-07-14 通解架構)**:不是「零 shim」(A2/A4 連續時間本質不可達,Gemini 亦告誡保留
  shim 架構),而是「**零手寫特例**」—— 終態沒有一行認站點的代碼,shim 清單收斂為「自動衍生機制 +
  三個資料檔(netlist-patch / analog-sidecar / timing-calibrations)」。
- **對外敘事**:S1a 是「二值開關級 → 混合訊號」的最小擴充實證。它回答 `semantic-ceiling.html` 提出的
  「未來可能提出什麼機制」——這份文件就是那個答案的藍圖。**能做出來最好;做不出來,這份機制盤點與
  島成本分析本身就是可發表的負面/邊界結果**(呼應整個專案「可證偽的負面結果」精神)。
