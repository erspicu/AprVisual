# 優化歷程總表 — 已採用的 win 與死路清單(2026-06-21 整理)

> 這份是把散在記憶裡的「熱路徑優化結果」整合成一張可瀏覽的對照表。
> MEMORY.md 只留一行索引指到這裡;單項細節(commit、配對勝場、安全性推理)在各記憶檔(recall 載入)。
> 規則:event-driven 單核已近天花板,real-time(42.95M hc/s)不可達;新點子預設 ≤1% 或負,**加成本必輸、唯有移除成本才贏**。

## 一、量測規則(先看)

- **這台 Zen2 不要鎖頻**(使用者 2026-06-12:鎖頻恢復後電腦會怪、要重開機)。Zen2 用「不鎖頻 interleaved-paired / round-robin + 中位數 + 配對勝場 + 同日同 session」。
- **sub-1% A/B 的權威基準 = Pi5 A76 鎖頻 3.0GHz**(真 governor lock、無 SMU 漂移)。Zen2 難分勝負時以 Pi5 為準。
- **一次只能跑一個 benchmark**,其餘 CPU 保持安靜(別多代理同時測)。
- bit-exact 是硬門檻:先過 mapped-checksum(golden `0x9174E19D961CB6E5` @400k `--extra-ram`;smoke `0x794A43ABDF169ADA` @300k)才看速度。
- `--pin`(affinity/priority/EcoQoS)允許(不碰頻率);batched A/B 不可信(時間漂移偽造方向)。
