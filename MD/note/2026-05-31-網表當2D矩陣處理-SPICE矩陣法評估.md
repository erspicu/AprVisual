# 把網表當「2D 矩陣」處理可行嗎?—— SPICE / 矩陣法 vs 我們的事件驅動 BFS

> 2026-05-31 的討論筆記。問題:「有沒有人把網表建構成 2D 矩陣的概念下去處理?主要是相依性問題很難處理。」
> 結論先講:**有,而且很成熟 —— 那就是 SPICE / MNA(導納矩陣)。矩陣法確實讓相依性(SCC、順序)問題消失,但它把成本換成「每步解整個系統」,正是 switch-level 當年被發明來逃離、而我們也實測過更慢(oblivious 121×)的東西。** 對「離散 + 動態稀疏」的我們,事件驅動圖走訪才是對的工具;矩陣全解是反方向。

## 1. 「電路 = 2D 矩陣」就是 SPICE / MNA
類比電路模擬用 **Modified Nodal Analysis**:把電路寫成 n×n 導納矩陣 `G`,解線性系統 `Gx = b`(稀疏 LU),每 timestep 解一次。
**相依性問題在這裡「消失」**:LU 不在乎環(SCC)、不在乎順序,所有節點耦合一次解完。—— 這個直覺是對的,這正是矩陣法的賣點。

## 2. 但這正是 switch-level 被發明來「逃離」的東西
Bryant 的 switch-level 模型(我們用的)當年就是為了**避開 SPICE 的矩陣成本**:把電晶體抽象成開關(on/off),用離散 group-resolution 取代連續矩陣解。原因:
- 稀疏 LU 每步 ~O(n^1.x–n²),而且**每半週期解一次整個 14.7K 節點系統**。
- 這等於「每步重算所有節點」= oblivious 全掃描。我們實測:**oblivious 121× 慢、AOT-batch 3-6× 慢** —— 因為丟掉了「平均只有 ~1.4 個節點變動」的事件稀疏性。

**所以這是 trade,不是免費午餐:矩陣讓你「不必追相依性」,代價是「每步解整個系統」。** 對離散 + 動態稀疏的問題,事件驅動圖走訪在漸進與實務上都遠勝矩陣全解 —— 你覺得難的相依性,用事件處理其實比用矩陣解掉還便宜。

## 3. 成熟的折衷版:FastSPICE(且已被我們的 IR 路涵蓋)
工業界 FastSPICE(NanoSim / CustomSim)= 「矩陣 + 事件驅動」折衷:把電路切成 **channel-connected-component(CCC)小區塊**、每塊各自矩陣/查表解、區塊間事件驅動。對我們:
- 為**類比連續精度**設計,對離散邏輯是殺雞用牛刀。
- 「切區塊」撞 **94% 一個大 SCC**(切不開,變一個巨塊)。
- 「抽 CCC 各自解」= 正是我們測過 **3-6× 慢的 IR/codegen 路**。

## 4. 我們其實已經在做「矩陣的離散稀疏版」
- 扁平 transistor list **就是稀疏鄰接矩陣(CSR 風格)** —— 不是沒用矩陣表示,是沒在上面做線性代數。(密集 2D 矩陣 = 14.7K² ≈ **2.16 億格**,直接撐爆 L1d,不可能。)
- settle-to-quiescence **本質是離散版 Gauss-Seidel relaxation**(矩陣迭代解法的離散形)。
- → 我們做的是「矩陣 relaxation 的離散、事件驅動、吃稀疏性」版本,對這個 regime 就是對的版本。

## 5. 唯一「2D 矩陣天生適合」的地方:規則陣列
矩陣 / 2D-grid 真正贏的場景是**結構規則**的子電路:
- **RAM / ROM / 暫存器** —— 已經是行為層(handler)處理,不當電晶體跑(等於已 2D-array 化)。
- **解碼 PLA**(6502 decode ROM 本身就是 2D AND/OR 矩陣)—— 唯一「當位元遮罩矩陣評估」可能贏過 transistor BFS 的具體候選。**但**它屬 IR-fringe(抽子塊行為化)、輸出又回饋進 SCC(邊界棘手),屬守則管轄範圍 → 真要試,先手寫 PLA-as-bitmask 原型、量「快過 S1 + bit-exact」再說;預期收益不大。

## 誠實結論
**有人做、且很成熟(SPICE / FastSPICE);矩陣法確實讓相依性消失,但成本換成「每步解整個系統」,正是 switch-level 設計來逃離、我們也測過更慢的東西。** 結構性瓶頸是「離散稀疏 vs 連續密集」的不匹配,不是缺一個矩陣技巧。規則陣列(RAM/ROM/暫存器)我們已用 array/handler 處理掉;剩下唯一候選是 PLA(IR-fringe,按守則先驗證最小原型,預期 sub-few-%)。
