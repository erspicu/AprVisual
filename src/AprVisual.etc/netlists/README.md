# netlists/ — visual6502 raw netlist data (other CPUs)

Per-chip raw [visual6502](http://www.visual6502.org) netlist definition files, copied here
from `ref/visual6502-master/` so AprVisual.etc can validate / benchmark non-NES chips.
This is the data side of the cross-CPU study — see
`MD/note/2026-06-15-其他CPU網表對照_前置研究.md`.

## Layout (one directory per chip)

```
netlists/
  6502/   segdefs.js  transdefs.js  nodenames.js   (visual6502 root files)
  6800/   segdefs.js  transdefs.js  nodenames.js   (ref/visual6502-master/chip-6800/)
  z80/    segdefs.js  transdefs.js  nodenames.js   (ref/visual6502-master/chip-z80/)
```

A future `--netlist <dir>` (or reuse of `--system-def-dir`) load-path parameter will point
the engine at one of these directories. **Note the format gap:** the current engine
(`Sim/WireCore.Parse.cs`) loads *MetalNES `.js` module* definitions (`data/system-def/nes-001`
etc.), **not** these raw visual6502 `var segdefs = [...]` arrays. A raw-netlist loader is the
next piece of work; these files are staged here for it.

## Format (raw visual6502)

- `segdefs.js`  — `var segdefs = [ [node, pull('+'/'-'), layer, x1,y1, x2,y2, ...], ... ]`
- `transdefs.js`— `var transdefs = [ [name, gate, c1, c2, bbox[4], geom[5], weak], ... ]`
  (6800 / z80 carry the 7th `weak` boolean, like 2A03/2C02; NMOS, c1/c2 interchangeable)
- `nodenames.js`— `var nodenames = { name: nodeNumber, ... }`

## Why the data is git-ignored here

These `.js` files are **not committed** (see `.gitignore` in this directory): they are licensed
third-party data, mirroring the repo policy that keeps all of `ref/` out of version control
(`CLAUDE.md`: "*licensed data / large third-party projects … don't copy code/data out of it
without checking its licence*"). Only this README + the directory structure (`.gitkeep`) are
tracked. To repopulate a working copy, copy the three files per chip from the `ref/` paths above.
(If you decide the NC-SA terms are acceptable to redistribute in this public repo, remove the
`*.js` rule from `.gitignore`.)

`support.js` / `testprogram.js` (JS sim helpers, not netlist data) were intentionally not copied.
