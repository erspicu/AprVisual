### A. The Exact Mathematical Condition & The Zero-Cost Invariant

Let $\mathbb{D}$ be the join-semilattice of drive flags, where the join operation $\sqcup$ is the bitwise OR. Your priority decode $\rho : \mathbb{D} \to \{0,1\}$ determines the concrete state.

When an event toggles the gate of transistor $t=(u,w)$, we want an exact $O(1)$ condition to skip enqueueing $u$ and $w$.

#### 1. The Turn-ON Theorem
**Event:** $t$ closes. $C_{new} = C_u \cup C_w$. 
**Condition to Skip:** `if (s[u] == s[w]) return;`
**Proof:** Because $\rho$ operates on the commutative idempotent OR of the components, joining two components that currently evaluate to the same state $V$ will *always* result in a component that evaluates to $V$. (The maximum of two drivers that both resolve to $V$ also resolves to $V$; similarly, joining two floating components of state $V$ yields a floating component of state $V$). Thus, **no state changes can occur**.

#### 2. The Turn-OFF Theorem
**Event:** $t$ opens. $C_{old}$ splits into subgraphs. We know $s[u] = s[w] = V$ because they were previously equipotential.
When does $C_{new} \subseteq C_{old}$ evaluate to $\neg V$? 
By the subset property of the semilattice, $\Lambda(C_{new}) \sqsubseteq \Lambda(C_{old})$. 
- To resolve to `1`, $C_{new}$ **must** contain at least one 1-driver (`Pwr`, `SetHigh`, `PullUp`, `State`). 
- To resolve to `0`, $C_{new}$ **must** contain at least one 0-driver (`Gnd`, `SetLow`, `ForceCompute`). 
*(Note: A floating component inherits the previous state $V$, so it never spontaneously flips to $\neg V$).*

**The $O(1)$ Invariant (Fixing Attempt #4):**
Instead of maintaining expensive dynamic driver *counts* per node, we maintain a static **Abstract Domain Over-Approximation**: `group_flags`.
Whenever your current solver pops a node, rebuilds the component $C$, and evaluates $\Lambda_{old} = \bigsqcup_{v \in C} flags(v)$, **write $\Lambda_{old}$ to an 8-bit field `group_flags` on all members of $C$**. 
*Crucially, writing this byte happens in the exact same loop that writes the state $s(v)$, touching the identical cache line. The maintenance cost is essentially zero.*

**Condition to Skip Turn-OFF:**
```c
uint8_t lambda = group_flags[u]; // The flags of the entire C_old
if (s[u] == 0) {
    // State is 0. It can only change to 1 if C_old contained a 1-driver.
    if ((lambda & MASK_1_DRIVERS) == 0) return; // Provably no-op
} else {
    // State is 1. It can only change to 0 if C_old contained a 0-driver.
    if ((lambda & MASK_0_DRIVERS) == 0) return; // Provably no-op
}
enqueue(u); enqueue(w);
```
**Proof of Exactness:** If $C_{old}$ lacked a 0-driver, no subset $C_{new}$ can contain a 0-driver. A component with no 0-drivers either contains 1-drivers (resolving to 1) or is floating (holding the previous state 1). Either way, it resolves to 1. The chaotic fixpoint is mathematically guaranteed to remain identical.

You noted that 94% of the influence digraph is one giant SCC, meaning you cannot statically schedule it. However, the simulation currently uses **Gate-Centric Propagation**: whenever a node changes, it blindly propagates to the channels it gates, forcing structural re-evaluations.

### C. The Theoretical Equivalence

This solution maps directly to three established mathematical theories:

1. **Abstract Interpretation (Widening)**: We want the exact fixpoint in the concrete domain (the exact graph $G(s)$). Attempting to maintain the exact graph is too expensive. Instead, we map the graph to an abstract domain (the Powerset of Drive Flags $\Lambda$). The abstraction function is $\alpha(C) = \bigsqcup_{v \in C} flags(v)$. Because your priority decode $\rho$ is bounded by this abstract domain, we can perform a $O(1)$ widening check. If the abstraction proves no change is possible, we safely halt evaluation.

2. **Idempotent Semiring (Tropical) Shortest Paths**: In switch-level theory (e.g., Bryant's MOSSIM), state evaluation is an algebraic path problem on a semiring of $(state, strength)$. Finding the value of a node is equivalent to finding the maximum-strength path to a driver. It is a known property of dynamic max-priority trees that *edge additions* are $O(1)$ local checks, but *edge deletions* require re-growing the tree. By caching the semiring sum of the entire component ($\Lambda_{old}$), we bypass the count-to-infinity problem and avoid the tree-regrowth overhead for deleted edges that don't isolate the node from the winning strength.

3. **Monotonic Hold-Previous**: The capacitive tie-break is notoriously non-monotonic. The non-monotonicity is analytically contained, allowing the abstraction to hold true.
