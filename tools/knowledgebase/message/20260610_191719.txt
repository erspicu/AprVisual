### 1. Where is the actual compute win? (And why S2 mostly whiffs)
The win in S2 would have to come from **(a) Node-COUNT reduction** (eliminating internal nodes = fewer events to process). 

It does *not* come from (b) cheaper per-node eval. Why? An S1 pure-logic node eval (read pull-down gates, LUT) is just a few masked loads and a random LUT read. An S2 gate eval (read input nodes, bitwise `~(A | B)`) replaces a LUT read with a bitwise op, but in a memory-bound, branch-unpredictable event loop, the ALU delta is swallowed entirely by the cache latency of fetching the inputs/events. 

It *could* come from (c) collapsing multi-transistor pass-logic (muxes/decoders) into gates, **BUT** in the 2A03/2C02, virtually all pass-logic networks rely on the isolated gate capacitance of their destinations to hold state (dynamic latches). Because you *must* preserve dynamic memory nodes (falling back to analog/BFS), you cannot collapse the pass-transistors without exposing the internal dynamic nodes, fracturing the "gate" back into single switches.

### 2. How much node reduction is actually on the table? (NMOS Reality Check)
**Almost none.** You asked what a realistic upper bound is. 
In CMOS, a 3-input NAND has 2 internal stack nodes. In NMOS, **series pull-down stacks are extremely rare** because the body effect and linear-region resistance severely degrade the $V_{OL}$ (logic low) level. 

Historically, 6502/NES designers avoided NAND/AOI structures, relying overwhelmingly on parallel pull-downs (NOR gates) and pass-transistor logic. 
*   A 4-input NMOS NOR has 1 output node and **zero** internal nodes. 
*   If your 3,929 pure-logic nodes are 90% NOR/Inverters, you have maybe 200–400 internal series-stack nodes chip-wide.
*   If you eliminate 400 nodes out of 14,723 (~2.7%), your event volume drops by roughly the same percentage. That is noise.

**What to count at load time:** Iterate your 3,929 pure-logic nodes. Count how many transistors are in their pull-down networks. Count how many distinct nodes are in those networks excluding GND and the output node. **That number is your exact absolute maximum node reduction for static gates.** If it's under 1,000, S2 is dead.

### 3. CCC → Gate Extraction for NMOS
If you insist on trying, standard EDA relies on **Series-Parallel Graph Reduction** (from Bryant’s COSMOS or ANAMOS).
1.  **Form CCCs:** Standard BFS over Source/Drain.
2.  **Filter:** Must have exactly 1 pull-up (depletion).
3.  **Graph Reduction:** Treat the Pull-Down Network (PDN) as a graph between Output and GND. 
    *   Find two transistors in series (internal node degree = 2). Replace with one edge, boolean = `A & B`. Drop the internal node.
    *   Find two transistors in parallel (shared S/D). Replace with one edge, boolean = `A | B`.
    *   Repeat until reduced to a single edge. The output is `~(PDN_boolean)`.
4.  **Pitfalls:** 
    *   **Bridge Circuits:** NMOS sometimes uses bidirectional pass-transistors *inside* a static pull-down network to save transistors (e.g., a barrel shifter or XOR). Series-parallel reduction fails here. You must fall back to path-sensitization (finding all valid paths to GND) or Binary Decision Diagrams (BDDs).
    *   **Pass-networks:** If a CCC has NO pull-up but connects to GND via pass gates, DO NOT EXTRACT. It is bidirectional pass-logic.

### 4. The Hybrid Boundary (Digital ↔ Analog)
This is where the nightmare starts. 
*   **Logic → Analog:** The logic node outputs a boolean `0` or `1`. In S1, this must translate to `SetLow` (strong GND) or `SetHigh` (strong VCC) so the BFS can resolve the analog CCC. This is cheap.
*   **Analog → Logic:** The analog node resolves via BFS to `VCC`, `GND`, or `Float`. The logic node evaluates. If the analog node floats, what does the logic gate see? 
    *   In the 6502, floating pass-nodes hold their previous charge (dynamic logic). You must ensure S1's floating-hold logic resolves the analog node to a hard `0` or `1` *before* the logic gate reads it.
*   **Event scheduling:** If a logic gate eval takes 0 half-cycles, but an analog BFS settles over a micro-tick, you introduce a race condition that didn't exist in S1's unified settle loop. You will have to queue digital-to-analog boundary events carefully, likely blowing up your event queue size and negating the node reduction.

### 5. Soundness of Behavioral Equivalence
What does switch-level S1 get right that S2 static gates get wrong?
*   **Glitch / Intra-settle races:** S1 naturally handles overlapping clock phases (e.g., $\phi_1$ and $\phi_2$ both high for a nanosecond) by forming a massive CCC and resolving the dominant drive (often resulting in a transient state that prevents a dynamic latch from flipping). Static logic will instantly eval a glitch, potentially destroying the state of a downstream dynamic node before the clock edge resolves.
*   **Threshold Drops ($V_t$):** NMOS pass transistors pass a "weak 1". A logic gate in S2 assumes inputs are rail-to-rail. If an S1 pass-transistor passes a weak 1 to a NOR gate, it might behave differently in edge cases than an idealized `~(A|B)` gate. (Though usually, Visual6502 ignores analog voltages anyway).

**The Pass/Fail Number:**
If the collapsible node count is **< 1,500 (10% of total)**, abandon S2. 
Why? Because Amdahl's Law is brutal. If S1 is memory-bound on event-driven BFS, eliminating 10% of the nodes eliminates roughly 10% of the events. A 10% reduction in queue pops will yield at best a 10% speedup. 115K hc/s goes to 126K hc/s. You need 42.95M hc/s.
